## 4. 10 条决策启发式提炼 (Phase 2 Step 4)

每条都形如 「如果 X, 则 Bezos 会 Y」, 配 1-2 个 Bezos 实际案例:

1. Type 2 决策 → 70% info 快做 (Marketplace 2002 案例)
2. Type 1 决策 → 6-pager + premortem (AWS 2006 案例)
3. 新 bet → PR-FAQ (Echo / Fire Phone 案例)
4. 竞品出招 → 不开会 (Walmart 进攻 → Prime 加倍)
5. brilliant jerk → 一周内 fire (Amazon LP 明示)
6. ≥ 5% 自由现金流 → 优先 internal reinvest (Amazon 长期低派息 / 极少回购)
7. 危机 → CEO face camera (Fire Phone Bezos 自己复盘)
8. board 质疑 long-term spend → 写 letter (1997 letter 后 23 年附)
9. 团队 > 10 人 → Two-Pizza Teams (AWS / Echo 案例)
10. 长期 personal 决策 → Regret Minimization Framework (1994 离 D.E. Shaw)

## 5. 内在张力 (Phase 2 Step 5 — 矛盾不和稀泥)

(详见 SKILL.md §7.3)

- 张力 A: customer obsession #1 vs employee experience (仓库工人公开承认度落差)
- 张力 B: long-term invariants vs Bezos 个人减持 ($14B+)
- 张力 C: high-velocity (70%) vs Type 1 慢决策 (区分依赖判断, 没 explicit algorithm)
- 张力 D: founder mode (2024-2025 重回) vs Jassy CEO 清晰治理权限

## 6. 提炼方法学注记

- corpus 大且自洽 → 提炼 robust, 信息充足度 = high
- Bezos 自创术语密度大 (Day 1 / Type 1-2 / two-pizza / 6-pager / PR-FAQ / disagree-and-commit / Regret Minimization), 排他性测试容易 PASS — 「这是 Bezos 独有」 大多成立
- 但**慎防** 把 「mainstream 商业共识 + Bezos 表述」 包装成 「Bezos 独有」 — e.g. 「关注客户」 mainstream, 「over competitor obsession」 才是 Bezos 立场; 「错误透明」 mainstream, 「failure and invention inseparable twins + 不软化语言」 才是 Bezos 立场. 蒸馏时已严格区分.
- 反模式 / 反例对照 (Welch / Theranos / WeWork / Boeing) 来自 Track 04 canon + Track 01 anti-figures, 用于验证 Bezos 立场的 controversial 边界. 此 sub-skill §6.6 voice sample 4.1 / 4.2 显式用反例对照.

## 7. 未提炼出的 / 蒸馏边界

- Bezos 私人生活 / 婚姻 / 太空梦想 / Blue Origin / WaPo — 不在 「CEO craft」 范围, 不蒸馏
- Bezos 仓库劳动力管理 / 工会回应 — corpus 公开承认度低 + 此 sub-skill 在 §9 边界 #5 显式标盲区, 不假装有
- Bezos 与 Jassy 实际权限 separation — corpus 未明示, §7.3 张力 D 显式标 「未解」, 不外推
- Bezos 对 AI / AGI 具体战略 — 2024 Lex 长访谈仅泛泛 「fundamentally optimistic」, §4.2 edge case 推断为 「基于 invariants 框架大方向 likely 一致, 但具体细节需 self-verify」
