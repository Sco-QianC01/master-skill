#### 维度 1: 战略与定位语境 (Strategy & Positioning Context)
- 看什么: 公司当前 stage (0→1 / 1→100 / mature / turnaround); 行业结构 (5 forces 当前状态); own 7 Powers 哪几条成立; 是否处于 SIP (10x 力 + 客户行为转向 + 一线吐槽).
- 在哪看: 致股东信近 5 年 invariants 段; 行业分析 (Stratechery / Acquired); 客户访谈 ≥ 10 次; 一线员工 skip-level 1:1.
- 输出: 1-page 战略 (winning aspiration → where to play → how to win → capabilities → mgmt systems) + 列出哪 1-2 Power 是 own moat + 列出 ≥ 1 个潜在 SIP 信号.

#### 维度 2: 资本配置基线 (Capital Allocation Baseline)
- 看什么: 当前 5 levers 金额分布; own hurdle rate; 近 3 年同类公司 capital allocation 决策 outcomes; own free cash flow 与 invariants 一致性.
- 在哪看: 10-K + 财报 + Buffett / Bezos / Mark Leonard 致股东信; Mauboussin hurdle rate 框架; comparable companies database.
- 输出: 5 levers 当前分布表 + own hurdle rate 测算 + 列出近 12 月 ≥ $X 的资本配置 thesis + 下一 24 月预期分布建议.

#### 维度 3: 高管班子健康度 (SLT Health & Talent Density)
- 看什么: SLT 平均任期; keeper test 通过率; brilliant jerk 在 SLT 比例; 文化抗拒者在 SLT 比例; A-player 比例; SLT 1:1 cadence + content quality.
- 在哪看: SLT 季度 360 + 外部 Spencer Stuart / Korn Ferry assessment; 自我观察 SLT 1:1; 跨级 skip-level 1:1; employee engagement (Lattice / Culture Amp / 心声社区).
- 输出: SLT 健康度 scorecard (talent density / culture fit / 任期分布 / keeper test 通过率) + 列出未来 3 月需 hire 的关键岗位 + 列出可能需要 transition 的高管.

#### 维度 4: 决策结构与节奏 (Decision Structure & Cadence)
- 看什么: 近 12 月 Type 1 决策数量 (应 ≤ 5, 业内估 Bezos / Mulally / Nadella 范式 T04-S053 T04-S035 T04-S012); 每个 Type 1 是否有 6-pager + premortem + post-mortem; SLT 是否真 Disagree and Commit (沉默反对率); WBR red/yellow 比例约 10% 以上 (业内估).
- 在哪看: decision journal 近 12 月; WBR red/yellow 历史; SLT 1:1 中 disagreement 频率; post-mortem 文档.
- 输出: 决策结构 audit (Type 1 数量 / 6-pager 比例 / premortem 覆盖率 / WBR red transparency) + 列出 3 个最近 Type 1 决策的执行健康度.

#### 维度 5: 长期 invariants 与 bets 平衡 (Invariants vs Bets Balance)
- 看什么: own 公司 5-10 invariants 是否 explicit + 5 年内变化率约 20% 以内 (业内估, Bezos / Buffett letters 范式 T04-S055 T04-S073); 当前 active bets 列表 + 风险分布; 致股东信 invariants 段是否 anchor 投资人.
- 在哪看: 历年致股东信 invariants 段对比; bets 内部 OKR + 资本配置历史; bets 失败的公开复盘 (透明度).
- 输出: invariants 列表 (5-10 条) + bets 当前 active 表 (含 Type 1/2 标注 + reversibility) + 致股东信对应段建议.

#### 维度 6: 文化与 mistakes 浮现机制 (Culture & Surface-Mistakes Mechanism)
- 看什么: 文化文档 (Culture Memo / LP / 心声社区) 是否 explicit + 持续重申; 文化抗拒者 fire 比例; mistakes 公开机制 (致股东信 mistakes 段 / WBR red / 心声社区匿名); psychological safety 信号 (员工敢说 bad news 比例).
- 在哪看: 文化文档版本史; fire 数据 + cause 分类; 致股东信 mistakes 段; 内部匿名渠道 + CEO 真去 read 记录.
- 输出: 文化健康度 audit (文化抗拒者 fire 率 / mistakes 公开质量 / psychological safety 信号) + 列出 3 个最近 公开 mistakes 案例.

#### 维度 7: 治理与利益相关者 (Governance & Stakeholder Map)
- 看什么: board composition (Lead Director / Audit Chair / Comp Chair 独立性); board off-site 频率 + 质量; 维权 surveillance (13D/13F monitor); shareholder primacy vs stakeholder capitalism own 立场 + ESG 披露义务 (按地); 是否有 activist 信号.
- 在哪看: proxy filing (DEF 14A); board pack 历史; Lead Director 月度 1:1; ISS / Glass Lewis engagement; 13D/13F filing monitor.
- 输出: 治理 audit (board composition + activist signal + ESG 披露状态) + 列出 next 12 月预期的治理变动 (e.g. director election / proxy / activist 风险).

> 跨学派共识 + 三重验证 (跨场景复现 ≥ 2 子场景 + 跨人物 ≥ 2 figures + 排他性). 每个模型 ≥ 2 处 source_id evidence + 局限. 三重 PASS / PASS / PASS or PARTIAL 通过, FAIL 降级 §2 playbook.

### 1.1 资本配置 = CEO 最高杠杆工作 (Capital Allocation as the #1 Job)

> (figures: Warren Buffett / William Thorndike / Mark Leonard / Jeff Bezos / Michael Mauboussin / 段永平 Duan Yongping)

- **一句话**: 「CEO 一辈子要做的几十个决策中, 资本配置 (5 levers: 内部再投 / 并购 / 派息 / 回购 / 还债) 的复利结果决定企业 20 年后的市值排序; 不算 hurdle rate / IRR 等于不会做 CEO」
- **应用方式**: 任何 ≥ 5% 自由现金流的去向决策, 先在 5 levers 同台用 own hurdle rate (best alternative return, 不是 WACC) 比较, 不预设答案; 写 1-pager thesis 上 board; 在年度致股东信公开 own 逻辑让投资人长期理解
- **局限**: (a) 在创业早期 (现金流为负) 不成立, capital allocation 退化为「融资 + 烧钱节奏」; (b) 在产品 / R&D-heavy 行业 (NVIDIA / Tesla 早期), capital allocation 重要但不是「唯一」最高杠杆 — Bezos 自己说过 「allocation 重要但不是 sole」; (c) 中国 A 股 + H 股回购受 regulation 限制, 西方 buyback playbook 不能照搬
- **evidence**: [T01-S017, T01-S019, T01-S046, T02-S015, T02-S016, T04-S015, T04-S073, T04-S074, T04-S075]

### 1.2 战略转折点 + 主动 paranoia (Strategic Inflection Point + Paranoia as Discipline)

> (figures: Andy Grove / 任正非 Ren Zhengfei / Jensen Huang / Jeff Bezos / Satya Nadella / Ben Horowitz)

- **一句话**: 「10x 力 (技术 / 竞争 / 监管 / 用户行为) 改变行业基本面时, CEO 必须最先识别 + 主动毁掉一部分老业务穿越 SIP (Intel 1985 退 DRAM / 任正非 1998 华为冬天 / Nadella 2014 mobile-first cloud-first) — Day 2 公司 stasis, then irrelevance, then excruciating decline, followed by death」
- **应用方式**: 听一线 (1:1 + skip-level, 不靠中层过滤); 用 5 forces / 7 Powers 重新评估 own moat; 主动给自己制造小危机 (季度 SIP audit + 「我们公司能怎么死」premortem); 出现 SIP 信号时公开站队不假装中立, 留 1-2 个老 guard 退路但 leadership 全部 wartime mode
- **局限**: (a) Paranoia 过度 → 组织慢性焦虑 + 创新麻痹 (晚期 Grove 自承); (b) 在稳态行业 (公用事业 / FMCG 龙头) SIP 频率低, paranoia 应稀释为「年度 review」非「每月制造危机」; (c) 与 §1.6 长期主义有内在张力 — 短期 paranoia 不能侵蚀长期 invariants
- **evidence**: [T01-S007, T01-S008, T01-S013, T01-S014, T01-S034, T04-S007, T04-S012, T04-S038]

### 1.3 写作 = 思考 + 文件 / 仪式 / 节奏是 CEO 最高杠杆工具 (Writing is Thinking; Documents / Rituals / Cadence > SaaS)

> (figures: Jeff Bezos / Peter Drucker / Warren Buffett / Mark Leonard / Patrick Collison / 任正非)

- **一句话**: 「CEO 最高杠杆的工具不是任何 SaaS, 是 5 件「文件 / 仪式 / 节奏」: 致股东信 + WBR + 1:1 + Town Hall + 6-pager; 不能用 narrative 写清楚战略 = own 没想清楚 — Amazon 16 年禁 PPT 内部决策, 强制 6-pager」
- **应用方式**: 重大决策不开 deck 会, 开 6-pager silent reading 会 (30 min 沉默读 + 提问 + 决策); 战略 = 1-page narrative 而非 50-slide deck; CEO 自己写致股东信不外包 PR; weekly cadence 设计 (周一 WBR / 周三 1:1 / 周五 Town Hall) 是公司操作系统的骨架; 仪式失守 = 治理失守先兆
- **局限**: (a) 在 < 50 人公司 6-pager 是 overhead, 创业团队应短信 / 白板 / 站会; (b) 不写英文的 CEO (中文 / 日文圈) 翻译损失 narrative 严密性, 需要更长篇 + 更具体案例补偿; (c) 写作能力是 CEO 的瓶颈门槛 — 不擅长写的 CEO 必须 hire ghostwriter 但承担「机构化沟通失真」风险
- **evidence**: [T01-S001, T01-S017, T01-S046, T02-S001, T02-S003, T02-S009, T02-S082, T04-S001, T04-S004, T04-S023, T04-S053, T04-S055]

### 1.4 高管班子 + Talent Density > 一切流程 (Executive Team + Talent Density > Process)

> (figures: Reed Hastings / Andy Grove / Ben Horowitz / Patrick Lencioni / Lou Gerstner / Bill Campbell)

- **一句话**: 「CEO 约 30-40% 时间在 SLT (7-12 人) 工作上 (业内估, Spencer Stuart CEO Life Cycle 业内综述); 1 个 A-player 顶 3 个 B-player; talent density 高之后才能 context-not-control; 容忍 brilliant jerk 是 CEO 第一管理罪 — 业绩 OK 但文化烂的 SVP, fire 越早损失越小」
- **应用方式**: (a) 每季度 keeper test 全 S-team (「如果他明天离职, 我会不会全力挽留? 不会 → 主动 fire + 高 severance」); (b) hire 用 cross-functional bar raiser 防 inflation; (c) fire 一周内 done 不拖; (d) 文化抗拒者即使 high performer 也 fire (Gerstner / Nadella 共同选择); (e) CEO 自己也需要 keeper test — 外部 coach (Bill Campbell 范式) + 独立董事 + 360 诚实自评
- **局限**: (a) 「不解雇低 + 高 severance」前提是高现金流公司 (Netflix 范式), 现金流紧张时不可行; (b) Keeper test 强度依赖文化 — 在中国 / 日本 lifetime employment 期待 culture 下需大幅软化; (c) Talent density 高的代价是 「招聘门槛高 = 招聘慢」, 早期超快扩张时矛盾
- **evidence**: [T01-S010, T01-S021, T01-S025, T01-S048, T02-S013, T02-S019, T02-S048, T04-S008, T04-S010, T04-S019, T04-S021]

### 1.5 Type 1 vs Type 2 决策 + 70% 信息快做 + Disagree and Commit (High-Velocity Decisions)

> (figures: Jeff Bezos / Daniel Kahneman / Gary Klein / Annie Duke / Andy Grove / Ray Dalio)

- **一句话**: 「不可逆 (Type 1) 决策与可逆 (Type 2) 决策不对称: Type 2 错了便宜 (回退就好), Type 1 错了昂贵 (lock-in); 大多数决策是 Type 2, 应授权 + 鼓励 70% 信息快做; 仅 Type 1 (定收购 / 关业务 / 换 CEO) 用 6-pager + premortem + disagree-and-commit 全套程式」
- **应用方式**: (a) 决策开始时 explicit 宣布 「这是 Type 1 / Type 2」; (b) Type 2 默认下放, CEO 不参与; (c) Type 1 写 6-pager 含 base rates + comparable cases + premortem (90 min 想象「2 年后已失败, 反推为什么」); (d) S-team 充分 disagree 表达 (沉默反对是最大叛变); (e) CEO 决策 + 全员 disagree-and-commit; (f) reversibility window + post-mortem 90 天后明确写入 decision journal
- **局限**: (a) 区分 Type 1/2 本身需要判断 — 错把 Type 2 当 Type 1 → 慢决策 + 流程过重; (b) Premortem 在小事上是 overhead; (c) 中国 / 日本 consensus culture 下 「disagree-and-commit」 需更长 disagree 时间窗口, 不能照搬 Amazon 范式
- **evidence**: [T01-S001, T01-S006, T01-S053, T02-S001, T03-S076, T04-S005, T04-S031, T04-S051, T04-S053]

### 1.6 长期 invariants + 短期 bets 分离 (Long-Term Compounding +守 5-10 Invariants 不变)

> (figures: Jeff Bezos / Warren Buffett / 任正非 / 王兴 Wang Xing / 张一鸣 Zhang Yiming / 段永平 Duan Yongping / 稻盛和夫 Kazuo Inamori)

- **一句话**: 「20 年复利的秘密是守住 5-10 个 invariants 不变 (Amazon: customer obsession / lowest price / fast shipping / wide selection) 同时持续 try 100 个 bets (Fire Phone / Echo / AWS); CEO 用 letter / town hall / 全员信持续 anchor invariants — 不变 才让 变 有意义」
- **应用方式**: 每年致股东信明确列 invariants 与最近 bets; bets 失败公开复盘绝不软化 (Bezos Fire Phone / Buffett Dexter Shoes); invariants 季度被挑战时主动 push back; 关键决策追问「这违反 invariant 吗? 如违反, 我们要不要修改 invariant?」(很少); 反例: 「数字化转型」「AI-first」这些是 bet 不是 invariant, 别错混
- **局限**: (a) Invariants 写得太抽象 = 无约束; 写得太具体 = 锁死创新; (b) 中国创业公司 0-3 年 invariants 还在演化, 强行锁定有害; (c) 「长期主义」被滥用为「不要被衡量」借口 — Bezos 自己说过 「不是 long-term excuse for no short-term metric」
- **evidence**: [T01-S001, T01-S017, T01-S046, T01-S053, T01-S055, T01-S063, T01-S067, T04-S038, T04-S055, T04-S073, T04-S074]

### 1.7 Mistakes 公开是信任货币 + 让坏消息浮上来 (Acknowledge Mistakes; Let Bad News Surface)

> (figures: Warren Buffett / Jeff Bezos / Reed Hastings / Alan Mulally / Ray Dalio / 任正非 (灰度 + 自我批判))

- **一句话**: 「CEO 不认错 = 系统无学习 + 投资人折现信任 + 员工学会隐瞒 — Buffett 每年致股东信详细列 own mistakes 60 年, 任正非 把『自我批判』机构化, Mulally Ford BPR red/yellow 不被惩罚反受 help; 沉默反对 + 隐瞒 red 是 CEO 最贵的盲区」
- **应用方式**: (a) 致股东信 mistakes 段不外包 IR; (b) 危机沟通先 acknowledge + 自陈, 不软化语言 (Tylenol 范式 vs Boeing 反面); (c) WBR red 受 help 不受罚 (Mulally Ford 范式); (d) 心声社区 / 内部匿名渠道 + 真去 read (任正非 30 年范式); (e) 用 base rate + 历史数据反问 own SLT 「我们为什么会被打脸?」; (f) CEO 季度自我 keeper test, 写入 decision journal
- **局限**: (a) Public 公司 mistakes 公开可能引发 SEC / shareholder lawsuit, 法务 review 必要但不能用作 shield; (b) 「让坏消息浮上来」前提是 talent density + psychological safety, 缺这两条 mistakes 公开 = political weapon; (c) 中国 context 政治敏感 mistakes 公开需考虑监管语境, 内部诚实可以更外部需谨慎
- **evidence**: [T01-S013, T01-S017, T01-S025, T01-S050, T02-S005, T02-S010, T02-S073, T04-S014, T04-S031, T04-S035, T04-S073]

## 标准 Playbook

> 形如「如果 {场景}, 则 {决策方向}」, 每条配 1-2 个具体案例 + ≥ 2 处 source_id evidence. Phase 3 cli_writer 会自动按 cluster 抽成 cli/decision/{cluster}.sh 决策树脚本.

1. **如果你刚当上 CEO (founder 或职业 CEO 都适用) 在头 100 天**: 则按 30-30-30 节奏走 — 第一 30 天 listen + diagnose (1:1 直到 top 50 都见过 + read 财报 + customer call ≥ 10 次, 绝不宣布战略); 第二 30 天 articulate own theory of business (1 页文档与 board + S-team 对齐); 第三 30 天启动第一波 mechanism (weekly cadence + S-team goals + 第一波 hire/fire 信号). 案例: Lou Gerstner IBM 1993 头 90 天著名拒绝 reporters 「I don't have a strategy yet」 (T04-S014); Satya Nadella Microsoft 2014 culture diagnose listening tour 前 6 个月 (T04-S012). evidence: [T03-S060, T04-S012, T04-S014]

2. **如果决策是不可逆 (Type 1)**: 则走 6-pager + premortem + disagree-and-commit 全程 (写 narrative memo 含 base rates / comparable cases / premortem 反推失败原因, S-team 充分 disagree, CEO 决断, 全员 commit, 90 天 post-mortem 写入 journal). 案例: Amazon AWS 2003 决策 (T02-S001); Microsoft 2014 mobile-first cloud-first 战略转向 (T04-S012). 反例: WeWork 2018 IPO 仓促推进未 premortem 撤回 (T01-S089). evidence: [T01-S001, T03-S068, T04-S031, T04-S051, T04-S053]

3. **如果决策是可逆 (Type 2)**: 则授权下放给 SLT 或下一层, CEO 不参与, 鼓励 70% 信息快做, 错了快回退. 案例: Bezos 1997 致股东信 「most decisions are reversible — make them quickly」 (T04-S055); Jensen Huang 60 直接下属 + 不开 1:1 让信息流速达极致 (T01-S032, T02-S065). evidence: [T01-S001, T01-S032, T04-S053]

4. **如果出现 ≥ 2 个 SIP 信号 (10x 力 + 客户行为转向 + 一线员工反复吐槽)**: 则 CEO 主动毁掉一部分老业务穿越 (Intel 1985 退出 DRAM 模板) 而非保护; 公开站队 + 写公开信 + leadership 全部 wartime mode + 留 1-2 old guard 退路. 案例: Intel 1985 退 DRAM (T04-S007); 任正非 1998 「华为的冬天」内部信 (T04-S038); Microsoft 2014 mobile-first cloud-first (T04-S012). evidence: [T04-S007, T04-S012, T04-S038]

5. **如果遇到 high-stakes 高管 hire / fire**: 则 hire 用 cross-functional bar raiser 防 inflation, fire 一周内 done 不拖; 文化抗拒者即使 high performer 也 fire (Gerstner / Nadella 共同选择); 高 severance + 公开 + 不让 lingering (Horowitz Hard Things 第 5 章). 案例: Hastings Netflix keeper test (T04-S010); Gerstner IBM 头一年 fire 全 senior team 大部分 (T04-S014); Horowitz Opsware 解雇高管 detailed playbook (T04-S008). evidence: [T01-S010, T04-S008, T04-S010, T04-S014]

6. **如果 ≥ 5% 自由现金流去向决策**: 则在 5 levers (内部再投 / 并购 / 派息 / 回购 / 还债) 同台用 own hurdle rate (best alternative return) 比较不预设答案; 写 1-pager thesis 上 board; 年度致股东信公开决策逻辑让投资人理解 own 框架. 案例: Buffett 60 年致股东信资本配置范式 (T04-S073); Mark Leonard Constellation Software 致股东信 hurdle rate 实操 (T04-S074); Henry Singleton Teledyne 1970s 反向 buyback (T04-S015). 反例: GE Welch 短期 EPS engineering buyback 长期破坏复利 (T04-S044). evidence: [T01-S017, T01-S046, T04-S015, T04-S044, T04-S073, T04-S074]

7. **如果危机 / 黑天鹅事件 (产品安全 / 数据泄露 / 治理曝光)**: 则按 30min / 1h / 24h / 72h / 1 week / 30 day 节奏 (30 min 内召集 crisis team / 1 小时内对外初步声明含 acknowledge + investigate + commit / 24h 内首份事实更新 / 72h 内 root cause hypothesis / 1 周内 corrective action plan / 30 天内 lessons learned 公开 / 长期 governance 改造). 案例: Tylenol 1982 Burke 7 天召回范式 (T03-S067); 反例: Boeing 737 MAX 2018-2024 Calhoun 否认 + 慢响应 (T01-S092, T03-S071). evidence: [T03-S067, T03-S068, T03-S071]

8. **如果 board 关系 / 致股东信 / 投资人沟通**: 则 board 不是汇报对象是 monitor + advisor — 设 quarterly review + 偶尔 deep-dive 教育 board on industry; Lead Director 1:1 月度 own first detect activist 信号; 致股东信每年 own 亲笔不外包 PR; 不靠 stock buyback 推 EPS — 长期 invariants 才是 anchor. 案例: Bezos 25 letters 1997-2020 持续 anchor invariants (T04-S055); Patrick Collison Stripe Press 致股东信范式 (T01-S043); Mark Leonard 致股东信 (T04-S074). evidence: [T03-S063, T04-S025, T04-S055, T04-S074]

9. **如果跨阶段切换 (founder→scaling / scaling→mature / mature→turnaround)**: 则诊断当前 stage (头数 100/500/2000/5000 + 文化 / 现金流 / 增速); 识别需要切换信号; 评估 founder 能否切换; 必要时 hire COO 过渡或 founder→executive chair; 重写 OS (组织架构 + meeting cadence + capital allocation 全部刷新); 90 天 transition plan + 12-24 月稳态评估. 案例: Brian Chesky Airbnb 2020 founder mode 重启 — fire 1900 员工 + scrap brand + skip-level + product cadence 全部 founder hands-on (T01-S035, T01-S037); 任正非 形式上让 CEO 轮值但 spirit 持续 founder mode (T01-S013); 反例: 任 Kalanick Uber 2017 founder unable to scaling → 被赶下 + Khosrowshahi 接 (T01-S091). evidence: [T01-S035, T01-S037, T03-S070, T04-S070, T04-S071]

10. **如果文化 / 价值观决策 (culture as OS)**: 则文化不是写在墙上的 values, 是 hire-and-fire 决策的累积; talent density 高之后才能 context-not-control; 文化抗拒者必 fire; 文化文档 (Netflix Culture Memo / Amazon LP / Huawei 心声社区) 是「契约 not perks」; 必须 explicit + 持续重申 (Town Hall + 致全员信) + 高管 model behavior. 案例: Hastings Netflix Culture Memo 2009 (T01-S026); Amazon Leadership Principles 16 条 (T06-S002); 任正非 心声社区 + 自我批判机构化 30 年 (T02-S005); 反例: WeWork community / Uber 早期 hustle culture 失控 (T01-S089, T01-S091). evidence: [T01-S025, T01-S026, T02-S005, T02-S020, T06-S002, T06-S007]

## 工具栈与选型决策树

> 直接消化 Track 02 (CEO 工艺的「工具」= 文件 / 仪式 / 节奏 + 少数 SaaS). 三层结构, 每层至少 3 件, 按行业惯例 (Section 1) 跨学派共识标记.

### 3.1 必备工具 (8 件, 约 80% CEO 都用 — 业内估, 跨 Track 02 + Track 04 公开材料综合)

1. **致股东 / 致全员信 (annual letter)** — Bezos / Buffett / Mark Leonard / Patrick Collison / 任正非 / 张一鸣 公认 mechanism. CEO 亲笔 + 一致性 > 文采 + 不变量 + 错误自陈. evidence: [T02-S001, T02-S003, T02-S016, T02-S082]
2. **WBR (Weekly Business Review)** — Amazon / Mulally Ford / Grove Intel 范式; 50-80 个 metric / red-yellow-green / red 受 help 不受罚. evidence: [T02-S006, T02-S010]
3. **1:1 (one-on-one)** — Grove HOM 1983 原创 + Bill Campbell 范式 (Trillion Dollar Coach); 高管周度 1:1 + skip-level 月度. evidence: [T02-S006, T02-S047]
4. **Town Hall / All-hands / AMA** — 全员沟通 + Q&A; 匿名问题机制. evidence: [T02-S021, T02-S043]
5. **6-pager (Amazon narrative memo)** — silent reading 30 min + 提问 + 决策; 替代 PPT. evidence: [T02-S009, T02-S023]
6. **OKR cascade (Objectives + Key Results)** — Grove Intel iMBO → Doerr 普及 → Google / Stripe 标配; 季度 cadence; CEO O = 公司 north star. evidence: [T02-S007, T02-S008]
7. **Board meeting + board pack** — quarterly + 24h 前 pre-read + executive session; Lead Director 月度 1:1. evidence: [T02-S026, T02-S028, T02-S063]
8. **Earnings call (上市公司)** — quarterly + script + Q&A; CEO + CFO + IR 三角合作; CEO 主战略叙事 + CFO 主数字. evidence: [T02-S032, T02-S033, T02-S034]

### 3.2 场景特化 (按 stage / 行业 / 治理形态选, 8 件)

1. **Carta / Pulley / Ledgy (cap table + 409A + 期权管理 SaaS)** — 早期 + 私募阶段必装; 上市后转 EQUITY EDGE / Computershare. 适用: pre-IPO; 不适用: 50+ 年大上市公司. evidence: [T02-S022, T02-S024, T02-S025]
2. **Diligent Boards / Nasdaq Boardvantage / OnBoard (board portal)** — 上市公司 board pack 加密分发 + voting + executive session 留痕. 适用: 上市 / 高治理要求; 不适用: < 50 人公司. evidence: [T02-S026, T02-S028, T02-S029]
3. **ISS / Glass Lewis (proxy advisor 双寡头)** — 上市公司 say-on-pay + director election engagement; 维权防御. 适用: 上市公司; 不适用: 私募. evidence: [T02-S030, T02-S031]
4. **SEC EDGAR + 10-K / 10-Q / 8-K / 14A (上市披露)** — US 上市公司核心披露; CSRC 信息披露管理办法 (中国); HKEX listing rules (港股). evidence: [T02-S032, T02-S033, T02-S034]
5. **Persefoni / Watershed / Sweep (climate / ESG dashboard)** — EU CSRD 2024 + SEC 2024 climate rule 触发的合规工具. 适用: EU 销售 ≥ €150M / 美国上市公司; 不适用: 私募 + 小公司. evidence: [T02-S036, T02-S037, T02-S038, T02-S071]
6. **Lattice / 15Five / Culture Amp (performance + engagement SaaS)** — 50-5000 人公司高管 review + 1:1 cadence + employee NPS. evidence: [T02-S039, T02-S040, T02-S041]
7. **Workday / Loom (HCM + async video)** — 大公司 HR + 异步全员沟通; CEO 用于 board pack walkthrough. evidence: [T02-S042, T02-S043]
8. **EOS Traction / Scaling Up / Rockefeller Habits (small-company OS)** — 小公司 ( < 250 人) 1-page 战略 + L10 weekly + quarterly rocks. 反例: 过度扩展到 1000+ 人 staff 时机械化失人情. evidence: [T02-S061, T02-S062]

### 3.3 新兴工具 (近 12 个月出现, 未必稳定, 8 件 — 都标 high decay)

1. **AI 战略 copilot (OpenAI o1 / Claude Opus 4 / Gemini 2.5)** — 2025 主流, Reid Hoffman + Dharmesh Shah 倡导; CEO 用作 thought-partner 写战略草稿 / 反向论证 / base rate 抓取. evidence: [T05-S001, T05-S023]
2. **AI 会议准备 (Granola / Fellow.ai / Otter.ai)** — 录音 + 自动摘要 + 行动项; 改变 1:1 + WBR 节奏. evidence: [T05-S010, T05-S031]
3. **异步 board 治理 (Slack DM + Loom video + Diligent async vote)** — 后疫情 founder 圈; 公开公司仍以同步为主. evidence: [T02-S043, T02-S026]
4. **NLP CEO 360 评估 (Workday Peakon / Culture Amp AI insights)** — 自动从 review 文本抓 sentiment; 用于 board CEO 评估. evidence: [T02-S041]
5. **维权 surveillance (Diligent Activism Solution / ISS Activism)** — 自动监测 13D / 13F filing + 早期 detect activist. evidence: [T02-S030]
6. **Founder letter SaaS (Beehiiv + Substack 用作内部信)** — 高频 founder mode 内部传播. 反例: 频率过高 = 噪声. evidence: [T05-S001, T05-S002]
7. **AI thought-partner for decision memo (Claude Projects / GPT-4 长文档)** — 写 6-pager 时反向论证 + premortem 抓 base rate. evidence: [T05-S001]
8. **DAO 治理实验 (Coinbase Commerce / Aragon, 反例)** — 2017-2022 web3 founder 圈试; 大多数失败回归 traditional governance; 列入但标 anti-pattern. evidence: [T05-S023]

### 3.4 选型决策树 (按 stage + 行业 + 治理 5 个分支)

- **stage = early (< 50 人, < $5M ARR)**: 致股东信 + 周度 standup + Carta + Notion → 跳过 board portal / proxy advisor / ESG dashboard / EOS
- **stage = scaling (50-500 人, $5M-100M ARR)**: 加 OKR cascade + WBR + 1:1 cadence + Lattice + 月度 board meeting → 跳过 ESG / 维权工具
- **stage = mature private ($100M-1B ARR)**: 加 6-pager + Town Hall + Diligent Boards + Persefoni (如 EU 销售)
- **stage = 上市公司**: 全套 + ISS engagement + SEC filing tooling + earnings call infrastructure + climate dashboard (按地)
- **stage = turnaround / wartime**: 临时增加 BPR-style daily review + 致全员信 high cadence + crisis comms agency; 暂停 quarterly OKR (cadence 缩到 weekly)

### 3.5 避坑清单 (≥ 5 条)

1. **错把 SaaS 当 OS**: Diligent Boards 不让你成为好董事会, ISS engagement 不让你成为好 CEO — 工具放大 + 留痕, 不替代 judgement
2. **stage 错位**: 50 人公司用 Diligent / ISS 是 overhead, 上市公司用 Sheet 当 cap table 是 negligence
3. **致股东信外包 PR**: 让 IR / PR 代笔 = 失去机构构建力, 投资人 5 年后能看出 own 是否一致
4. **WBR 报喜不报忧**: red/yellow 被惩罚一次, 下次全 green; Mulally Ford 范式是 red 受 help 不受罚
5. **founder mode SaaS 化**: Beehiiv 高频内部信 ≠ founder mode; founder mode 是 skip-level + deep-dive + 客户对话, 不是工具
6. **OKR 当 KPI 用**: OKR 是 stretch + ambitious, 完成 约 70% 是常态 (Doerr Measure What Matters T02-S007 共识业内估); 当 KPI 用 → 团队 sandbagging
7. **6-pager 写成 deck 转 PDF**: 6-pager 是 narrative + 论证, 不是 bullet list

## 工作流 / Pipeline

> 直接消化 Track 03 (8 workflow, 每个 ### N. 整数编号 + inline **入门 SOP**: numbered list, cli_writer.py 抽取格式). 资深路径 (skip + optimize + add) ≥ 2 类 + 近期变化 ≥ 1 处.

### 4.1 战略与年度计划制定 (Strategy & Annual Planning)

- **入门 SOP**: 1. 重读 Drucker theory of the business 三问 (谁是客户 / 客户认为什么有价值 / 我们的能力); 2. 召集高管 off-site 2-3 天; 3. 用 Lafley/Martin choice cascade 框架 (winning aspiration → where to play → how to win → capabilities → mgmt systems); 4. 输出 1-page 战略; 5. 翻译为公司 OKR cascade; 6. 高管会签字 + 致股东信草稿; 7. 季度滚动校准.
- **资深路径**: 跳过 SWOT (太基础); 优化 — 用 7 Powers 反向论证 own moat; 额外 add scenario planning + premortem「我们怎么死」; 额外 add base rate 检查 (同类历史中位数).
- **近期变化**: 2024 起 AI 让 base rate 抓取从人工调研变 2 小时; 2024 起 EU CSRD + SEC climate rule 强制把 ESG / 气候作为战略必含项; founder mode 2024 essay 让多家中型创业 CEO 重新 audit 「我是不是太 delegate」.
- **典型耗时**: off-site 2-3 天 + 后续 6 周打磨; 季度滚动 0.5 天.
- **关键人物**: Bezos (T01-S001) / Drucker (T04-S001) / Lafley & Martin (T01-S070) / Andy Grove SIP (T04-S007).

### 4.2 资本配置决策 (Capital Allocation)

- **入门 SOP**: 1. 列 5 levers 当前金额; 2. 算每条 own IRR 与 hurdle rate; 3. 写 1-pager thesis (含 base rate + comparable cases); 4. 提交董事会审议; 5. 公告投资人; 6. 季度复盘; 7. 年度致股东信公开复盘逻辑.
- **资深路径**: 跳过 「平均分」(直觉是 5 levers 平摊, 资深人 know 集中下注); 优化 hurdle rate 不用 WACC 用 best alternative return; 额外 add dis-synergy 检查 (买后 2 年, 协同失败 own 还是赚?); 额外 add 反例池 (近 10 年同类大额并购失败率 约 50%, 业内估 — McKinsey/Harvard M&A retrospective 综述).
- **近期变化**: 2024 起 AI 让 dcm / comp company 抓取从一周变 2 小时; 2025 起 利率 normalization 让回购 hurdle rate 抬高.
- **关键人物**: Buffett (T01-S017) / Thorndike Outsiders (T04-S015) / Mark Leonard (T01-S046) / Mauboussin (T04-S075).

### 4.3 高管班子搭建 / hire / fire (Executive Team Building)

- **入门 SOP**: 1. 写 JD + scorecard; 2. Korn Ferry / Spencer Stuart shortlist (5-10 人); 3. 多轮面试 + 案例 + 360 reference (≥ 5 人); 4. talent 密度评估 (keeper test); 5. 90 天 onboarding (offsite + 1:1 weekly + objectives); 6. 季度评估; 7. 必要时果断解雇 (一周内 done).
- **资深路径**: 跳过 senior hire 用 internal-only search (约 90% 失败, 业内估 — Korn Ferry/Spencer Stuart CEO succession data 综述 T01-S099 T01-S100); 优化 用 cross-functional bar raiser (Amazon 范式); 额外 add culture interview (Gerstner / Nadella fire 文化抗拒者); 额外 add 「解雇会 + 高 severance + 公开 + 不让 lingering」.
- **近期变化**: 2024 远程 / 混合 hire 比例上升; AI assistant 帮 review 1500+ resumes; 360 评估 用 AI sentiment.
- **关键人物**: Hastings (T01-S025) / Horowitz (T01-S021) / Lencioni (T01-S101) / Bill Campbell (T01-S048).

### 4.4 董事会与投资人沟通 (Board & Investor Relations)

- **入门 SOP**: 1. 季度前 2 周准备 6-pager 主题; 2. 24h 前发完整 board pack; 3. board meeting 不读 ppt 而是 silent reading 30 min; 4. 讨论 + Q&A; 5. executive session (CEO 离场); 6. 记录 + 行动项 assignment; 7. 半年度致股东信; 8. Lead Director 月度 1:1.
- **资深路径**: 跳过 100+ slide deck (改 6-pager); 优化 board off-site 1.5 天 (年度 + 半年度); 额外 add Lead Director monthly 1:1; 额外 add 致股东信 mistakes 段不外包.
- **近期变化**: 2024 起 SEC 4-day cyber 8-K + climate rule 让 board 议题更重; 2025 founder mode revival 让 founder-CEO 更频繁 challenge board orthodoxy.
- **关键人物**: Bezos 25 letters (T04-S055) / Larcker/Tayan (T04-S025) / Buffett (T01-S017).

### 4.5 WBR / 节奏运营 (Weekly Business Review & Operating Rhythm)

- **入门 SOP**: 1. 选 50-80 个 metric (north star + leading indicator + 安全 metric); 2. 周一 8am WBR (CEO + SLT + 数据 owner); 3. 异常深挖 (red/yellow); 4. action item assignment + owner; 5. 周三 staff 1:1 sweep; 6. 周五 all-hands; 7. 月度 OP review; 8. 季度 QBR.
- **资深路径**: 跳过 「都标 green」(资深人 know 没有 100% green 是正常的, 业内估 Mulally BPR 经验 T01-S072); 优化 red 受 help 不受罚 (Mulally Ford BPR); 额外 add base rate 比较 (同类公司 metric 中位数); 额外 add skip-level 1:1 抓 ground truth.
- **近期变化**: 2024 起 AI summary 让 1h WBR 缩到 30 min; 远程混合 cadence 趋稳定.
- **关键人物**: Bezos (T01-S001) / Mulally (T01-S072) / Grove HOM (T04-S005) / Jensen Huang 60 直接下属 (T01-S032).

### 4.6 重大决策 (High-Stakes Decisions)

- **入门 SOP**: 1. 判定 Type 1/2 (explicit 宣布); 2. Type 1 走 RAPID + premortem + base rate; 3. Type 2 快速 70% 信息决策; 4. 写 1-pager decision memo; 5. 异议时 Disagree and Commit; 6. 决策时点 + reversibility window 明确; 7. 90 天 post-mortem 写入 decision journal.
- **资深路径**: 跳过 「等所有信息」(资深人 约 70% 信息 + iteration 比 约 90% + 慢决策好, Bezos 2016 letter 范式 T04-S053 — 阈值为业内估); 优化 premortem session (90 min); 额外 add base rate (同类历史中位数); 额外 add S-team「我们最不愿被打脸的论点」预演.
- **近期变化**: 2024 起 AI 让 base rate / comparable cases 从 1 周变 2h; 2024 起 AI thought-partner 帮写 6-pager.
- **关键人物**: Bezos Type 1/2 (T04-S053) / Kahneman/Klein HBR 2010 premortem (T04-S051) / Annie Duke Thinking in Bets.

### 4.7 危机沟通 (Crisis Communication)

- **入门 SOP**: 1. 30 min 内召集 crisis team (CEO + COO + legal + comms + CFO); 2. 1 小时内对外初步声明 (acknowledge + investigate + commit, 不否认); 3. 24h 内首份事实更新; 4. 72h 内 root cause hypothesis; 5. 1 周内 corrective action plan 公开; 6. 30 天内 lessons learned 公开; 7. 长期 governance 改造; 8. 半年董事会复盘.
- **资深路径**: 跳过 「先 lawyer review」(资深 CEO 先 acknowledge 再 lawyer detail); 优化 CEO 本人 face camera 不让 PR 顶替; 额外 add pre-script 危机剧本 (Tylenol / Boeing / Wells Fargo 反例库季度演练); 额外 add board 高频沟通 (危机时 weekly board update).
- **近期变化**: 2023 SEC cyber 8-K 4-day rule; 2024 OpenAI board coup 让所有 SaaS founder pre-script「ousted by board」scenario.
- **关键人物**: J&J Tylenol 1982 (T03-S067) / 反例: Boeing Calhoun (T01-S092) / Wells Fargo (T02-S053).

### 4.8 跨阶段切换 (Stage Transition: 0→1 / 1→100 / mature / turnaround)

- **入门 SOP**: 1. 诊断当前 stage (头数 100/500/2000/5000 + 文化 + 现金流 + 增速); 2. 识别需要切换信号; 3. 评估 founder 能否切换; 4. 必要时 hire COO 过渡或 founder→executive chair; 5. 重写 OS (组织架构 + meeting cadence + capital allocation 全部刷新); 6. 90 天 transition plan + board buy-in; 7. 12-24 月稳态评估; 8. 必要时再切回 founder mode (Chesky 范式).
- **资深路径**: 跳过 「founder forever」教条 (Jensen / 任正非 / 张瑞敏 form 上不切但 spirit 切); 优化 hybrid mode (保留 founder 强项 — 产品 / 文化 / 客户 — cascade 其余); 额外 add 外部 advisor (Lou Gerstner / Brian Chesky 公开建议); 额外 add stage-specific OKR 重写.
- **近期变化**: 2024 Paul Graham「Founder Mode」essay 让 founder→pro CEO 不再是 unchallenged orthodoxy; 2025 Buffett→Greg Abel transition 设立 founder-to-pro 范式.
- **关键人物**: Brian Chesky Airbnb 2020 (T01-S037) / Paul Graham (T01-S036) / Lou Gerstner (T01-S038) / Reed Hastings 创始人→executive chair (T01-S025).

## 表达 DNA

CEO craft 的表达 DNA = 资深人聚一起讨论时的 register, 不模拟某 figure. 多人融合.

### 5.1 高频用语 / 黑话 (top 12)

Day 1 / Day 2 / 6-pager / 灰度 / Working Backwards / 人单合一 / 阿米巴 / Talent Density / Type 1 vs Type 2 / Hurdle Rate / Keeper Test / Founder Mode / Disagree and Commit / 长期主义 / 延迟满足 / 致股东信 / WBR / 心声社区 / 自我批判 / Wartime CEO / Strategic Inflection Point / Theory of the Business.

### 5.2 严肃 register

- 短句 + 数字 + 反例 ("we tried X in 2018, it failed because Y, here's what we learned")
- 反过度自信: "we don't know" / "我们也在摸索" / 「灰度」(任正非)
- 反 buzzword: 不说 「数字化转型」「赋能」「中台」「all in AI」 — 资深 CEO 用具体行动 (build the Echo team / fire the SVP) 描述决策
- 反第一人称: "the team built X" / "我们做了 X" 多于 "我做了 X"
- 引用 base rate + 历史: "the median acquisition fails 50% of the time, here's why ours is different"

### 5.3 内 vs 外沟通差异

- **对内 (Town Hall / 致全员信)**: 长 + 详细 + 含错误自陈 + 含 invariants + 含 bets; 任正非 内部讲话比公开访谈坦率 5x
- **对外 (致股东信 / earnings call)**: 长 + 详细 + 含 mistakes 段 + 含 invariants 段 + 反 「我们一切都好」; Bezos 25 letters 1997-2020 范式
- **对 board**: 6-pager + silent reading + executive session (CEO 离场 30 min)
- **对 activist**: 短 + 具体 + base rate + 不情绪化 (Trian / Pershing Square engagement playbook)

### 5.4 外行破绽 top 10 (这些一说出口资深 CEO 就 know own 是外行)

1. 说 「我们要做 X 行业的 Uber/Amazon/华为」(成熟 CEO 不会用「类比定位」)
2. 说 「先做 MVP 再迭代」(早期阶段词, CEO 阶段过了)
3. 把 KPI 当 OKR 用 (OKR 是 stretch, KPI 是必达)
4. 引用 Welch 9-box stack ranking 不带反思 (2022 后已是 anti-pattern)
5. 说 「996 是福报」 (法律违法 + 范式过时)
6. 说 「Move fast and break things」(Facebook 自己已修)
7. 把 「founder mode」简化为 「founder 永远不放权」(误读 PG essay)
8. 说 「我们要 customer obsession + employee 满意 + shareholder return + ESG」(everything for everyone = strategy for no one)
9. 用 「数字化转型 / 中台 / 赋能」无具体行动 backing
10. 把 Disagree and Commit 简化为「CEO 一票否决」(忽略 「充分 disagree 表达」前置)

### 5.5 对话样本库 (industry voice 实战语料)

#### 5.5.1 客户版 (面客 / 投资人 / 媒体长访谈)
- 「If we believe customers are our most important asset, then we should be willing to be misunderstood for long periods of time. ... We will continue to make investment decisions in light of long-term market leadership considerations rather than short-term profitability or short-term Wall Street reactions.」(source: T01-S001, Bezos 1997 letter 原话, 客户场景: 投资人质疑 Amazon 烧钱; 致股东信第一次 anchor invariants)
- 「我们公司不会有冬天吗? 不会! ... 你想过没有, 当我们公司有 10 万员工的时候, 我们的纪律会有多差? 我们的奋斗精神会有多差?」(source: T01-S013, 任正非《华为的冬天》1998 原话, 内部场景: 华为高速增长期主动 sound paranoia; 心声社区 公开)

#### 5.5.2 同业版 (私下 / 内训 / 同业访谈)
- 「Most decisions should probably be made with somewhere around 70% of the information you wish you had. If you wait for 90%, in most cases, you're probably being slow. Plus, either way, you need to be good at quickly recognizing and correcting bad decisions.」(source: T04-S053, Bezos 2016 letter 原话, 同业场景: 解释 Type 2 决策 + high-velocity decision making)
- 「only the paranoid survive ... business success contains the seeds of its own destruction. The more successful you are, the more people want a chunk of your business and then another chunk and then another until there is nothing left.」(source: T04-S007, Andy Grove Only the Paranoid Survive 原话, 同业场景: 给硅谷 CEO + Intel 高管 explicit 表达持续 paranoia)

#### 5.5.3 治理 / 投资人版 (公开场合谈治理 / 学术 / 资本配置)
- 「In the end, the work of capital allocation is the CEO's most important responsibility. ... Five potential capital allocation choices ... Each has very different consequences for shareholders.」(source: T04-S015, William Thorndike The Outsiders 转述 Henry Singleton 范式; capital allocation as #1 job)
- 「Charlie and I view the company's marketable securities as collateral that, like a savings account, provides liquidity for our operating businesses ... It's the boring, unglamorous work of capital allocation that compounds over decades.」(source: T01-S017, Buffett 致股东信 long-running 范式综述 转述; capital allocation discipline)

#### 5.5.4 反例版 (这一行的资深人**绝不会**这样说的话, 错位 / 被错位包装的销售话术)
- 「We're going to make this profitable in the next quarter by cutting all R&D and laying off engineering — the market will reward us for discipline.」(source: T04-S044 转述 of Welch revisited critique, 反例: 短期 EPS engineering 破坏长期复利, Welch GE 后期范式被 Gelles 2022 反思; 资深 CEO 不这样说)
- 「I'll figure out the strategy as we go — fake it till you make it, we just need to keep raising」(source: T01-S087 + T01-S096 转述 of Theranos collapse pattern, 反例: 系统欺骗 + 治理崩塌; Holmes 11 年判决; 资深 CEO 拒绝这种话术不论 stage)
- 「996 是我们公司的福报, 不愿意接受的可以走 — 客户需要我们」(source: T06 anti-pattern F.7 转述 of Jack Ma 阿里 2019 公开发言被广泛批评 + 中国最高人民法院 2021 996 违法判决, 反例: 工时崇拜 + 法律违法; 资深 CEO 灰度对待 工时但不公开美化 996)
- 「This M&A is going to unlock $5B in synergies — trust me, the integration is straightforward」(source: T06 anti-pattern F.3 转述 of AOL/Time Warner + HP/Autonomy 反例 pattern, 反例: acquisition theater, 不算 dis-synergy; 资深 CEO 写 1-pager 含 base rate「同类并购约 50% 失败率, 业内估」)

## 质量基准 + 反模式

### 6.1 什么算「好」 (5 条具体可验证基准)

1. **CEO 自己写的致股东信能与 5 年前对得上**: invariants 段 5 年内变化约 20% 以内 (业内估, Buffett / Bezos / Mark Leonard letters 跨年比较 T01-S017 T04-S055); 数字 / 案例可校对; 错误自陈段不是 PR boilerplate.
2. **board pack 是 6-pager 而非 100+ slide deck**: silent reading + 提问 + 决策结构; meeting 时间约 60% 以下用于过 deck (业内估, Working Backwards T02-S009 范式).
3. **WBR red/yellow 持续可见且未被惩罚**: 比例约 10% 以上 (业内估, Mulally Ford BPR T02-S010 经验); 全 green = 没人敢报 bad news = CEO 失去 ground truth; red 季度约 5% 以下是 paranoia 不足.
4. **Type 1 决策每年 ≤ 5 个, 都有 6-pager + premortem + post-mortem**: 数量太多 = 区分失败; 没有 6-pager + premortem = 决策结构失守.
5. **SLT 平均任期 ≤ 5 年 且离开是 either fire 或 「graduated」(自然下一阶段)**: > 5 年 = talent density 衰减; 全是被 fire = culture 失守; 全是 「graduated」 = 假装管理.

### 6.2 反模式 (10 条外行 / 入门常犯错误)

1. **Strategy as PowerPoint**: 战略不下沉到 OKR / capex / hire-fire, deck 漂亮但执行 0.
2. **Welch-style EPS engineering**: 短期 EPS 优化 buyback + 裁 R&D 破坏长期复利; Gelles 2022 反思.
3. **Acquisition theater**: 收购不算 synergy IRR; ego-driven 大额并购 (HP/Autonomy / AOL/Time Warner 反例).
4. **Slow firing**: fire 决定拖延让 cancer spread, talent density 衰减 + 文化污染.
5. **Tolerating brilliant jerks**: 业绩好文化烂的高管 CEO 不动, 一年内底下好人跑光 — Hastings keeper test 范式核心反对.
6. **Board surprise**: board 从媒体读到 own 公司新闻; Lead Director 1:1 缺位; 重大事件 not pre-briefed.
7. **PPT theater**: board pack 100+ slide 无真讨论, meeting 约 90% 时间过 deck (业内估 — Larcker/Tayan governance research T04-S025).
8. **Don't shoot the messenger 违反**: red 被惩罚一次, 下次全 green; Mulally Ford 范式公认反对.
9. **HiPPO bias + silent disagreement**: SLT 不发声 12 月后翻案; Disagree and Commit 没被建立成 norm.
10. **Lawyer-mediated denial + CEO ghost**: 危机时躲让 Comms 顶替, lawyer 先 review 拖延 acknowledge; Boeing Calhoun + Wells Fargo + BP Hayward 反例.

### 6.3 反面样本 (4 个不软化背书的 CEO 失败案例)

- **Elizabeth Holmes (Theranos)**: fake-it-till-you-make-it 11 年判决 (T01-S087 / T01-S096) — 警示: 「先假装成功后真成功」是欺诈, 不是 founder mythology
- **Adam Neumann (WeWork)**: 创始人神话 + 治理失守 + 自我交易 (T01-S089 / T01-S090) — 警示: founder mode ≠ 不受约束; cult of founder 是治理 anti-pattern
- **Travis Kalanick (Uber 2010-2017)**: 文化失控 + 性骚扰 + 法律 (T01-S091) — 警示: 「move fast」不能掩盖文化建设; talent density 不能容忍 brilliant jerk
- **Sam Bankman-Fried (FTX)**: 治理空白 + 25 年判决 (T01-S088) — 警示: charismatic founder + no board + no audit = systemic fraud

## 智识谱系

> 5 个主要流派 + 1 个 cross-school anchor (Bill Campbell 教练范式). 各派奠基 + 当前代表 + 核心分歧.

### 流派 A: Drucker-Effective-Executive 流派 (个人 OS + knowledge worker)
- **奠基**: Peter Drucker 「Theory of the Business」HBR 1994; Effective Executive 1966; Managing Oneself HBR 1999.
- **当前代表**: Drucker Institute; HBS Authentic Leadership Development; Bill George True North; 张瑞敏 (公开 ack 「Drucker 是我老师」).
- **特征**: 把 CEO 视为「靠决策与判断创造产出的脑力工作者」, 重个人 OS + self-management; 不强调 charisma 或 vision.
- **分歧**: Drucker 时代假设 organization man (单一组织 long tenure), 在 founder mode / startup / gig 时代部分前提需更新.
- **evidence**: [T04-S001, T04-S080, T04-S089, T04-S094]

### 流派 B: Operator-Engineer-CEO 流派 (Grove / Bezos / Mulally / Nadella)
- **奠基**: Andy Grove HOM 1983 + Only the Paranoid 1996; iMBO / OKR Intel 1971.
- **当前代表**: Bezos (Amazon 1997-2020 letters); Mulally (Ford BPR); Nadella (Microsoft Hit Refresh); John Doerr (OKR scaling); Brian Chesky (Airbnb founder mode 2020).
- **特征**: CEO = engineer + operator; cadence + mechanism > charisma; explicit framework + measurable output; 重 WBR + 6-pager + BPR.
- **分歧**: 在 hyper-creative / founder-driven context (Jensen Huang 反例) 过度 mechanism stifle; 也被批评机械化失人情.
- **evidence**: [T01-S001, T01-S072, T04-S005, T04-S007, T04-S012, T04-S035, T04-S053, T04-S055, T04-S071]

### 流派 C: Capital-Allocator-CEO 流派 (Buffett / Singleton / Thorndike / Constellation / 段永平)
- **奠基**: Henry Singleton (Teledyne 1960s+); Buffett 致股东信 1965-; Thorndike Outsiders 2012.
- **当前代表**: Mark Leonard (Constellation Software); Patrick Collison (Stripe Press 重印 Outsiders); Mauboussin; 段永平 Duan Yongping (中文 平行, value investing CEO).
- **特征**: CEO 单一最高杠杆是 capital allocation; 5 levers 优先级; hurdle rate > ROI; 极简 HQ + 分权 ops; 反 stack ranking + 反 buyback as EPS 工程.
- **分歧**: 是否适用 product / R&D-heavy CEO (Bezos 自己说 capital allocation 重要但 not 唯一); 中国 A 股回购作用受 regulation 限制.
- **evidence**: [T01-S017, T01-S046, T01-S065, T04-S015, T04-S073, T04-S074, T04-S075]

### 流派 D: Founder-Mode + Wartime-CEO 流派 (Horowitz / PG / Chesky / 任正非 部分对应)
- **奠基**: Horowitz Hard Things 2014 + a16z essays; Paul Graham Founder Mode 2024 essay.
- **当前代表**: Brian Chesky (Airbnb refound 2020); Reforge Founder Mode 课; 任正非 「华为冬天」+「灰度」(部分 spirit 对应).
- **特征**: 反 「founder graduate to manager」 orthodoxy; founder skip-level + deep-dive + 不 delegate 关键 + cross-stage 持续 founder leverage.
- **分歧**: founder mode 与 Uber Kalanick / Theranos Holmes 失控边界在哪? Bain Founder's Mentality 与 Khurana 「CEO 神话」批判是对立面.
- **evidence**: [T01-S035, T01-S036, T01-S037, T04-S008, T04-S009, T04-S070, T04-S071]

### 流派 E: 东方 CEO OS 流派 (任正非 / 张瑞敏 / 稻盛 + 张一鸣 / 王兴 / 黄峥 / 段永平)
- **奠基**: 稻盛和夫 (Kyocera 1960s+, 阿米巴 + 心性); 任正非 (华为 1987+, 灰度 + 自我批判); 张瑞敏 (海尔 1984+, 人单合一).
- **当前代表**: 张一鸣 ByteDance (延迟满足 + 理性算法); 王兴 美团 (长期有耐心 + 无限游戏); 黄峥 拼多多 (本分); 段永平 OPPO/vivo (本分 + 平常心).
- **特征**: 偏 认识论谦逊 (灰度) + 心性修炼 + 全员经营 (阿米巴 / 小微) + 持续守住 「本分」 + 警觉成功麻痹 (自我批判).
- **分歧**: 部分 founder-specific 不可移植 (任正非 个人 + 华为军工文化 contextual); 与西方 explicit framework 在 「decisiveness」(任正非 强势 vs 灰度) 表面矛盾 — 实质是哲学 vs operational 不同层.
- **evidence**: [T01-S013, T01-S053, T01-S055, T01-S063, T01-S065, T01-S067, T04-S037, T04-S038, T04-S039, T04-S041, T04-S042, T04-S104, T04-S105]

### Cross-School Anchor: Bill Campbell Trillion Dollar Coach
- 跨派 CEO 教练 (Apple / Google / Intuit), 教 CEO「人 + 1:1 + 信任 + community」, 不属任何派但所有派 CEO 私下都 ack 他对自己影响.
- **evidence**: [T01-S048, T01-S049, T01-S083, T02-S047, T04-S030]

### 核心分歧 (still-debated within the canon)
1. **Founder Mode vs Khurana 神话批判**: PG 推 founder skip-level + deep-dive; Khurana 警告 charismatic CEO 系统性 underperform — Phase 2 必须并存非二选一.
2. **Short-Term Buyback 反面 (Welch) vs Outsiders 回购正面 (Singleton/Buffett)**: 同一动作不同 context 截然不同 outcome — capital allocation 不是黑白.
3. **Welch-stack-ranking 反面 vs Hastings keeper-test (类似动作)**: 区别是 talent density + transparency, 不是动作本身.
4. **东方灰度 vs 西方 explicit framework**: 不是对立, 是哲学层 + 操作层不同; Phase 2 应作 「双系统并存」 而非 「二选一」.

## 诚实边界

> 7 条具体局限, 不模糊化, Phase 4 quality_check 会检.

1. **信息截止 2026-05-22**: §3 工具栈 + §4 工作流 (AI 战略 copilot / 异步 board / ESG dashboard) 衰减最快 — 建议每 3-6 月 update; 心智模型 + 智识谱系 1-2 年才动.
2. **中文 CEO canon 偏「个人讲话集 / 单一传记 / 海外学者写中国案例」**: 缺少 EN canon 那样「practitioner-written textbook」 — 任正非 没写过 textbook, 张瑞敏 没写过 textbook, 是讲话集汇编. 中文 CEO 思想的 systematicity 比 EN 弱; 提炼 mental_model 时中文 CEO 思想权重计算需谨慎 — 不是 「内容差」, 是 「形式差」.
3. **日文 CEO canon 极度偏稻盛和夫一人**: canon 约 85% 来自 Inamori (业内估, Track 04 canon 调研 T04-S104/S105); 张富士夫 / 柳井正 / 孙正义 systematic CEO writing 较少 → 日本 CEO 思想多样性在本 skill 不充分展开, 仅作约 5% 配 spice (业内估). 如用户场景明确为「日本 CEO」, 应触发独立扩展研究.
4. **幸存者偏差 + 时代红利**: 单个 CEO 的打法不是普适规律; 学术研究显示约 1/3 业绩 attributable to CEO, 约 2/3 是 sector / timing / luck / capital structure (业内估; HBR Spencer Stuart CEO Life Cycle 2024 研究综述). 任何 「学 Bezos / 学任正非」 都需 own context 重新评估 — 范式可借鉴, 全套照搬危险.
5. **public 一手内容失衡 (成功者发声多, 失败者沉默或被神化或被妖魔化)**: 8 个反面样本 (WeWork / Theranos / Uber-Kalanick / FTX / Welch revisited / Boeing Calhoun / Carlos Ghosn / Musk Twitter) 刻意纳入对冲此偏差, 但仍有大量「失败 CEO」公开内容稀薄.
6. **女性 + 非美 / 中 / 日 CEO underrepresented in canon**: 当前 retained 15 figures 中 Lisa Su (AMD) 是 retained 唯一女性 + 非美 CEO 之一; Indra Nooyi / Mary Barra / Karen Lynch / Christine Lagarde 等有 systematic 一手内容但未进 top 15 — 边界标. 用户场景明确为「女性 CEO」「印度 / 印尼 / 拉美 CEO」 应触发独立扩展研究.
7. **数字 / 法规快变项 — 业内估**: §3 工具栈 ESG / 气候 / 维权 部分 (CSRD 2025 Omnibus 可能修订 / SEC climate rule 部分 stayed / 中国公司法 2024-07 新法 / Trump 2.0 政府可能 rollback Dodd-Frank 部分条款) 现状预计 6-12 月内变动, SKILL.md 用户使用时需 self-verify 最新监管状态. 任何 % / 数字 (如 「约 1/3 业绩 attributable to CEO」「约 70% 来自 sector/timing/luck」) 都是业内估 + 公开学术综述, 非精确预测.
