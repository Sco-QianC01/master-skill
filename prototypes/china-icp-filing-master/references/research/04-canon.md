# Track 04 — Canon: China ICP Filing & Internet Content Compliance

中国网站备案 (ICP 备案) 与 互联网内容合规 这个行业的 canon 在所有 master-skill 蒸馏目标中天然结构最干净 —— 一手率高得反常, 因为它的源头从一开始就被国务院 / 全国人大 / 工信部 / 网信办 / 公安部 五条国家级权威主线垂直统治, 几乎不存在第三方解释能替代法规原文的情况。从 2000 年国务院令 292《互联网信息服务管理办法》起家, 一路到 2005 年信产部 33 号令《非经营性互联网信息服务备案管理办法》搭起 ICP 备案双轨, 再到 2017 年 CSL / 2021 年 DSL+PIPL+ 关基条例 / 2022-2023 年算法 + 深度合成 + 生成式 AI + 数据出境四件套, 主线全在 `*.gov.cn` 子域名一手登记, 不需要走任何 surrogate. 国家法律法规数据库 `flk.npc.gov.cn` 提供全国人大立法的权威全文版本, 工信部 `www.miit.gov.cn` + 备案系统 `beian.miit.gov.cn` + 网信办 `www.cac.gov.cn` + 公安部 `www.mps.gov.cn` 给出部委层面规章 / 通知 / 处罚通报. 因此 manifest 中 verified_primary 行占绝对主体, 这是这个行业的客观结构. surrogate_primary 主要分布在三个场景: (1) 学术专著, 必须经由 法律出版社 / 中国民主法制出版社 等 publisher listing 或 高校学者主页 (.edu.cn 实际上属 primary suffix, 故学者主页里有些直接 primary); (2) 律所网络法 / 数据合规 / 备案实务 alerts —— 中伦 / 君合 / 金杜 / 大成 / 海问 / 汉坤 等, 因为律所域名不在白名单 suffix; (3) 学术期刊 + 中国信通院等智库, 信通院域名 `.ac.cn` 不在 primary suffix, 需 surrogate. 反面 canon 收录滴滴 80.26 亿处罚案 cac.gov.cn 原文通报, 这是这个行业不可绕开的历史教训. 严禁微信公众号 / 知乎 / CSDN / 百度系 / 阿里云开发者社区即使做 surrogate 都不行 —— 因为法规一手在 .gov.cn 直接拿就够了, 无需第三方搬运.
