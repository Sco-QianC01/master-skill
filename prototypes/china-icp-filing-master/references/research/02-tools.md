# Track 02 — Tools: China ICP Filing & Internet Content Compliance

中国网站备案与互联网内容合规这条线的工具栈结构高度行政化, 不像 SaaS / AI 行业那样以"开源框架 + 商业产品 + 开发者生态"为骨架, 而是天然分裂为 **五大类不可互换的系统**: (1) **主备案监管系统** — `beian.miit.gov.cn` (工信部 ICP) + `www.beian.gov.cn` (公安部) + `beian.cac.gov.cn` (网信办算法/大模型) + 各省通管局子站, 全部为 .gov.cn 一手, 没有替代品也无法绕开; (2) **云厂商备案系统** — 中国独有, 法律层面 接入服务商 (云厂商 / IDC) 才是 ICP 备案的 **法定代提交方**, 因此阿里云 / 腾讯云 / 华为云 / 三大运营商 (天翼云 / 移动云 / 联通沃云) / 京东云 / 百度智能云 / UCloud / 金山云 / 青云 / 网宿 等 vendor docs 是 **流程性一手** 的唯一稳定入口, 全部以 surrogate_primary + "vendor docs" 标注; (3) **域名注册商系统** — `cnnic.net.cn` (.cn 域名管理机构) + 主流注册商 (西部数码 / 新网 / 易名 / 美橙 / 万网) 负责域名实名认证, 这是 ICP 备案的前置条件; (4) **等保测评 + 安全工具** — `itsec.gov.cn` (中国信息安全测评中心) + `djbh.net` (公安部等保网) + CNVD / CNNVD 漏洞库 + 主流测评机构 (启明星辰 / 安信 / 国信安 / 中国电科测评); (5) **法律检索数据库** — `flk.npc.gov.cn` (国家法律法规库, .gov.cn 一手) + 北大法宝 / 知网 / 万律 / 威科 (database surrogate) + 律所专题页 (君合 / 中伦 / 金杜 client alerts). **选型决策树** 主要发生在 云厂商 / 测评机构 / 律所 这三层 — 个人/中小企业首选 阿里云或腾讯云 (vendor docs 最详尽, 备案小程序流程化, 业内估市场份额 #1/#2 (本 prototype metadata)); 政企客户 / 国央企倾向华为云 + 三大运营商云 (合规背书与政府关系); 跨国企业首选金杜 / 君合 / 海问 + DLA Piper / Baker McKenzie 中国分所做 ICP 经营许可证申请; 等保三级测评必须找公安部认可名录的 60+ 家有资质机构 — 选择上 头部的启明星辰 / 绿盟 / 安恒 / 中国电科测评 与中小机构在测评深度与速度上差异显著 (业内估)。**严禁** 引微信公众号 / 知乎 / CSDN / 百度系 / 阿里云开发者社区 即使作 surrogate — 各厂商 vendor docs 均有官方 help / support / document 子域名替代。
