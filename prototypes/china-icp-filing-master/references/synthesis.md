# Synthesis — China ICP Filing & Internet Content Compliance

> Phase 2 — 提炼 6-track research 笔记为可运行的「中国网站备案 + 互联网内容合规 Master OS」。本提炼是 2026-05-18 截止的横切快照, 强政策属性 6-12 月内任何子条线 (算法 / 大模型 / APP 备案 / 数据出境 标准合同) 都可能有 1-3 次实质性微调; 心智模型 + 接入商责任体系 + 三横三纵框架 1-2 年才动, 是 master OS 的稳定层。

## 1. 心智模型 (Mental Models)

### 1.1 三横三纵立体备案框架

**一句话**: 任何一个跑在中国大陆境内的 网站 / APP / 小程序 / SaaS 都同时落在 三横 (ICP 备案 + 公安备案 + 等级保护测评) × 三纵 (主体 + 网站 + 经营性升级) 的九宫格里, 缺任何一格 都是不完整的合规。

**应用**: 接到「我要在中国上线一个 X 服务」首先 画九宫格 → 标出每格 现状 / 需要做的事 / 谁来做 (站长 / 接入商 / 法务 / 测评机构) / 时间线; 任何格子 标 "不清楚" 就先停下补做功课, 不要先 想 优化路径。

**局限**: 九宫格是 minimum scaffold, 实际 还有 行业特殊许可 (网络出版 / 网络文化 / 网络新闻信息 / 信息网络传播视听节目 / 互联网医院 / 教培 等) 是 第十/十一格, 视行业 加挂。

**figures**: 周汉华 (中国社科院) / 中国信通院 (政策与经济研究所) / 君合 cyber alerts
**evidence**: [T04-S001 / T04-S002 / T04-S031 / T05-S009 / T05-S033]

### 1.2 接入商责任体系 (中国独有制度)

**一句话**: 中国法定 ICP 备案的执行单位是 接入服务商 (云厂商 / IDC), 不是 申请人自己 — 站长不能直接向工信部提交, 必须 通过 阿里云 / 腾讯云 / 华为云 等接入商 完成 信息核验 + 提交 + 后续监测 + 异常上报。

**应用**: 选 云厂商 时, 备案能力 + 各省通管局对接经验 + vendor docs 详尽度 是和 算力 / 价格 同等重要的 三大决策维度; 中小厂商 (UCloud / 青云 / 金山云) 备案能力与 BAT 差距实质; 千万别 假设 海外 SaaS + 自己向工信部走通 — 制度不允许。

**局限**: 接入商责任 不等于 接入商替你扛违法 — 站长 提供 虚假资料 → 接入商 通过 不会 让站长免责 (但 接入商 承担 信息真实性核验 不力 的连带责任 + 各方都会被罚)。

**figures**: 阿里云备案 (help.aliyun.com) / 腾讯云备案 / 华为云备案 / 工信部令 33 号
**evidence**: [T02-S011 / T02-S012 / T02-S013 / T04-S002 / T05-S007 / T05-S008]

### 1.3 各省通管局执行口径差异 (30%+ 区域差)

**一句话**: 同一主体 + 同一备案材料, 上海 / 北京 / 广东 / 浙江 / 江苏 通管局的 审核严格度 + 通过率 + 反馈速度 与 二三线省份 差异 ≥ 30% (业内估), 同一省份 不同时期 (尤其 大型敏感事件后 30-90 天) 严 vs 松 也波动明显。

**应用**: 选 主体所属省份 时 (有 备案历史 的主体 跨省迁移 难, 但 新设主体 注册地有选择) 综合考虑 (a) 业务实际所在 (二者距离不能太远); (b) 同省接入商网络条件; (c) 该省通管局 历史 拒批率 + 反馈速度 (沪 / 京 严但快; 中西部 松但慢) — 不要 盲目 选 一线 不要 盲目 选 三线。

**局限**: 「严」与「松」是流动概念 — 各省 受 中央政策 + 省委 + 当年 重点工作 影响, 当前严松格局 6-12 月可能 因 中央专项 (例如 2024 重拳整治 短视频 / 2025 大模型审查) 翻转。

**figures**: 沪通管局 shca.miit.gov.cn / 京通管局 bjca.miit.gov.cn / 粤通管局 gdca.miit.gov.cn / 中伦 cyber alerts
**evidence**: [T05-S005 / T05-S006 / T02-S006 / T02-S007 / T02-S008 / T03-S001]

### 1.4 备案 ≠ 一次性工作 (持续合规属性)

**一句话**: ICP 备案 / 公安备案 / 等保测评 / 数据出境 / 算法备案 任何一项 都不是 拿到证就完事 — 主体信息变更 / 网站内容偏离 / 系统重大升级 / 安全员离职 / 数据跨境流向变化 都需 同步 重新走流程, 任何 长时间不维护 都会 在 工信部 / 网信办 / 公安部 抽查 中 暴露。

**应用**: 任何 中大型项目 备案后 必须 设立 持续合规岗位 (法务 + 合规 + 安全员 三角); 季度自查 + 年度 整改报告 + 重大事件 临时补办 是 标配, 不是 可选项; 把 备案 当 "上线门槛" 而非 "终身工作" 是 最常见且最危险 的误判。

**局限**: 持续合规 边际成本 对 个人 站点 / 小公司 偏重 — 此时 选择 简化业务模型 (避开 经营性 / 算法推荐 / 数据出境 等 高维护成本环节) 比 强行 全栈合规 更现实。

**figures**: 工信部抽查机制 / 国家市场监督管理总局 / 君合 cyber / 中伦 数据合规
**evidence**: [T04-S010 / T04-S017 / T05-S009 / T05-S011 / T05-S033]

### 1.5 备案合规 与 内容审查 双轨联动 (但 不等价)

**一句话**: 拿到 ICP 备案号 ≠ 内容可以随便发 — 备案 是 主体 + 网站 + 接入信息 行政程序; 内容 是 持续监管 + 实时审查, 内容违规 (政治敏感 / 涉黄 / 涉赌 / 算法滥用 / 直播带货不实 / 大模型有害输出) 直接 触发 ICP 注销 + 公安治安罚 + 严重 刑事, 即使 备案 完美 也救不了 内容违规。

**应用**: 任何 内容平台 (UGC / 直播 / 短视频 / AIGC / 论坛 / 评论) 备案 同时 必须 同步 内容审核团队 (机器 + 人工 + 24h 应急); 大模型服务 备案 + 内容标识 + 输出过滤 三位一体 缺一不可; 不要 把 "备案下来" 当 内容 free pass。

**局限**: 本 skill 严格 限定 在 备案 + 工程 + 法务 视角, 内容审查 的 具体价值判断 / 边界争议 不展开 — 这是 网信办 + 国家广电总局 + 国家电影局 等机构 + 业内 internal 编辑规范 的事。

**figures**: 网信办 cac.gov.cn / 网信办内容生态规定 / 算法推荐管理规定 / 深度合成管理规定
**evidence**: [T04-S010 / T04-S016 / T04-S025 / T04-S033 / T05-S002 / T05-S003]

### 1.6 监管联动 + 联合执法 (2023-2026 新趋势)

**一句话**: 单一监管机构 (只 工信部 或 只 网信办 或 只 公安部) 的 单点合规 已不够 — 2023-2026 工信部 + 网信办 + 公安部 + 国家市场监督管理总局 + 央行 + 中证监 + 国家广电总局 多机构 联合通报 + 联合执法 + 联合处罚 是 主流 (滴滴 80.26 亿罚 = 网信办主导 多机构 联动; 某 教培 整顿 = 教育部 + 网信办 + 工信部 + 国家市场监督管理总局 同步), 任何 master OS 都必须 跨机构 全栈 联动思考。

**应用**: 中大型项目 合规审计 必须 多机构 视角同步 — 备案合规 (工信部) + 内容合规 (网信办) + 网络安全 (公安部) + 反不正当竞争 + 反垄断 + 消费者权益 (国家市场监督管理总局) + 行业主管 (央行 / 教育部 / 卫健委 / 国家广电总局 视行业); 单一律所 + 单一法务 视角 已无法 覆盖。

**局限**: 多机构 联动 的 节奏 + 力度 受 中央政策 影响极大 — 不同时期 同 violation 可能 不同 机构 主导; 这部分 本 skill 给的 框架 是 截止 2026-05 状态, 未来 可能 重组 (新成立机构 / 机构 合并)。

**figures**: 网信办 cac.gov.cn / 工信部 miit.gov.cn / 公安部 mps.gov.cn / 国家市场监督管理总局 samr.gov.cn / 滴滴案例 CAC 通报
**evidence**: [T01-S041 / T01-S042 / T04-S041 / T04-S042 / T05-S002 / T05-S003 / T05-S004 / T05-S013]

### 1.7 行政许可 vs 行政备案 ≠ 同一概念 (程序门槛截然不同)

**一句话**: ICP 备案 / 公安备案 / 算法备案 / APP 备案 / 个人信息出境标准合同备案 是 行政备案 (备案制 — 申请 + 通过 + 上线; 不批就是 默示通过 视情况); ICP 经营许可证 / 网络出版服务许可证 / 网络文化经营许可证 / 网络新闻信息服务许可证 / 信息网络传播视听节目许可证 是 行政许可 (许可制 — 必须 实质审查 + 颁证; 不批 = 不能上线), 程序 + 时间 + 注册资本 + 人员 + 接入资源 门槛全部不同, 业内 长期 把这两词混用 是 最常见 法律错位。

**应用**: 第一步 永远 是 判断 你要做的事 落 备案 还是 许可 还是 两者都要 — 经营性 ICP (有偿 服务 / 撮合 交易 / 广告 / 内容付费) 一定 需 ICP 经营许可证 (许可); 非经营性 ICP (企业宣传 / 个人博客 / 公益) 只需 ICP 备案 (备案); 视频 / 直播 平台 通常 需 信息网络传播视听节目许可证 (许可 + 注册资本 1000w) 加 ICP 经营许可证 加 ICP 备案 (三证); 论坛 / UGC 通常 需 互联网新闻信息 / 网络文化 等 视情况。

**局限**: 部分行业 (新业态) 法规 滞后 (例: 早期 直播带货 / 早期 大模型 服务) — 监管 用 "适用 + 类推" 套老法规, 业内 短时间内 模糊 — 此时 须 看 网信办 + 工信部 当年 最新 通知。

**figures**: 国务院令 292 号 / 信部令 33 号 / 工信部 电信业务分类目录 / 君合 cyber alerts
**evidence**: [T04-S001 / T04-S002 / T04-S015 / T05-S009 / T06-S047]

## 2. 标准 Playbook (10 条 决策启发式)

1. **如果 服务向中国大陆用户 提供 网站 / APP / 小程序 / SaaS / 任何 IT 服务**: 则 ICP 备案 必备 — 个人 / 企业 / 事业单位 / 社会团体 通过 接入商 提交; 个人主体 严禁做 商业站, 企业主体 可做。 案例: 某 海外开发者 用 AWS US 部署中文站, 国内 DNS 污染 + 工信部抽查 强制关闭 (国内未备案 / 接入未在中国境内, 双重违规)。
2. **如果 服务 含 任一 收费 / 撮合交易 / 网络广告 / 信息付费 / 内容订阅 / SaaS 订阅**: 则 经营性 — 在 ICP 备案 之外 额外 申请 ICP 经营许可证 (B25 信息服务业务), 注册资本 跨地区 ≥ 1000 万 (本 prototype metadata) 或 省内 ≥ 100 万 (业内估), + 3 名社保 + 接入资源 + 工信部 60 工作日。 案例: 某 SaaS 仅 ICP 备案就开始 收费, 工信部抽查 强制 下线 + 罚款 + 6 月内 不许再申请。
3. **如果 ICP 备案下来**: 则 30 日内 必须 补办 公安备案 (www.beian.gov.cn, 不可拖延; 拖延 直接 公安部 通报 + 严重 关停)。 案例: 某中型电商 ICP 完成 6 月未办公安备案, 公安部网安总队 通报 + 限期 整改 + 责任人 行政罚 200。
4. **如果 系统 涉及 重要信息 / 公共服务 / 50 万 + 用户 个人信息 / 关基行业**: 则 等级保护 三级以上 测评 + 备案 必备, 找 公安部 认证测评机构 + 年度 复测 + 整改报告 (基准: GB/T 22239-2019)。 案例: 多家 在线医疗 平台 因 等保未到三级 被 卫健委 + 公安部 + 网信办 联合通报, 招投标资质 全国受限。
5. **如果 服务 含 算法推荐 / 个性化推送 / 内容排序**: 则 算法推荐备案 (网信办 国家算法备案系统) 必备, 同时 用户 反算法权 + 关闭 算法推荐 入口 + 标识 算法决策 三项 强制。 案例: 某 短视频 平台 未备案 算法 推送, 网信办 责令下架 + 罚款 + 公开整改通告。
6. **如果 服务 含 深度合成 (AIGC 文字 / 图像 / 视频 / 语音 / 数字人)**: 则 深度合成备案 + 内容显示 / 隐式 标识 双轨 必备, 同时 用户 知情同意 + 反深度合成 救济渠道。 案例: 某 数字人 直播 平台 因 未标识 深度合成, 被 网信办 通报 + 主播 + 平台 双重罚款。
7. **如果 服务 提供 生成式 AI / 大模型 服务**: 则 算法备案 + 服务备案 双轨, 训练数据 合法性 + 内容安全评估 + 安全员 + 24h 应急 + 公众沟通 全套, 上线前 至少 3-6 月 启动。 案例: 多家 大模型 创业公司 因 服务备案 未通过 上线 推迟 6-12 月, 烧钱周期延长。
8. **如果 个人信息 涉及 跨境 (出境 / 境外 处理)**: 则 三选一: (a) CAC 安全评估 (重要数据 + 100 万人 个人信息 + 关基运营者 + 累计 100 万 / 年 一律 须 评估); (b) 个保 标准合同 (中小型 SaaS / 跨国 集团内 默认路径); (c) 个人信息保护认证 (适合 较少 主体); 严禁 "先境内境外同步, 出事再说"。 案例: 滴滴 2022 数据出境违规, 网信办 + 工信部 + 公安部 + 国家市场监督管理总局 + 央行 + 国家税务总局 多机构 联动 罚 80.26 亿 + 主要负责人 100w (CAC 通报原文 2022-07-21)。
9. **如果 业务 主体 + 法定代表人 + 网站负责人 + 域名所有人 + 接入信息 任一变化**: 则 必须 通过 接入商 → 通管局 申请 变更备案, 不能 沿用 旧备案号; 跨省迁移 实际上是 注销 + 新设, 不是 简单变更。 案例: 某中型 SaaS 因 公司 名变更未及时 备案变更, 工信部 抽查 备案信息不符, 强制 注销 + 主体短期黑名单。
10. **如果 备案 被驳回 / 注销 / 关停**: 则 三层救济: (a) 接入商沟通 + 整改 + 重新提交 (90% (业内估) 案件最实际路径); (b) 各省通管局 行政复议 (60 日内, 形式审查为主); (c) 行政诉讼 (难度极高, 救济率低, 重点在 不能 立即 复活 业务); 关键 在 预防 + 不要 走到 第三层。 案例: 多家 互联网公司 备案被注销后 走行政诉讼, 平均胜诉率 ≤ 10% (业内估), 救济周期 6-18 月, 业务损失远大于 整改 + 重新备案 成本。

## 3. 工具栈 + 选型决策树

### 3.1 必备工具 (任何 中国境内 网站 / APP / SaaS ≥ 95% (业内估) 都要用)

- **ICP 备案系统** (beian.miit.gov.cn) — 主入口 + 查询 + 申报 (无替代)
- **公安备案系统** (www.beian.gov.cn) — 公安部 互联网安全管理服务平台 (无替代)
- **云厂商备案系统** (任一: 阿里云 help.aliyun.com / 腾讯云 cloud.tencent.com/document / 华为云 support.huaweicloud.com / 等) — 接入商代提交 必经路径

### 3.2 场景特化 (5 类, 按业务类型分)

- **若 业务 含 视频 / 直播 / AIGC 内容生成**: 加 算法备案系统 (网信办) + 大模型备案 + 深度合成备案 + 信息网络传播视听节目许可证 (视频) + 网络文化经营许可证 (直播)
- **若 业务 含 跨境 / 海外集团内数据共享**: 加 数据出境 三件套 (CAC 评估系统 + 个保 标准合同 + 个保 认证) + 国际律所 中国办公室 alerts (跨境视角)
- **若 业务 含 金融 / 支付 / 证券 / 保险**: 加 央行 + 中证监 + 国家金管总局 (前身银保监) 行业许可 + 数据本地化 + 反洗钱合规 + 行业 vertical 信息系统等保
- **若 业务 是 政府 / 央国企 / 关基行业 (能源 / 交通 / 水利 / 卫健 / 金融 / 教育)**: 加 关基条例 备案 + 数据出境 重要数据评估 + 强 等保 四级 + 国密 认证
- **若 业务 含 教育培训 / 在线教培 / K12 / 学科类辅导**: 加 教育部 + 国家市场监督管理总局 + 网信办 教培专项 + 学科类校外培训 监管 + 双减政策 联动 + 教培用户个人信息 强保护要求

### 3.3 新兴 / 实验工具 (近 12 月 出现, 未必稳定)

- 大模型 服务备案 + 训练数据备案 + 内容安全评估 体系 (2023-2026 在演变, 流程 + 模板 + 国家算法备案系统 UI 6-12 月 多次更新)
- APP 备案 工信部 2024-03 新规 (从 2024 年起 应用市场 全面下架 未备案 APP — 早期 部分代办 / 中小开发者 应对 不及)
- 数据出境 标准合同 备案 (2023 年起 简化路径, 中小型 SaaS 主流路径)

### 3.4 避坑清单

- **不要 用 developer.aliyun.com 当 阿里云备案权威源** — 它是 开发者社区 内容 (PR + 二手转贴); 阿里云备案权威源是 help.aliyun.com (vendor docs)
- **不要 把 ICP 备案系统 beian.miit.gov.cn 与 公安备案系统 www.beian.gov.cn 混用** — 名字相似 完全独立两套
- **不要 用 第三方备案查询 (icp.chinaz.com 站长之家) 当 准确数据源** — 用 beian.miit.gov.cn 官方查询
- **不要 选 黑代办 / 低价代办 / 假资料代办** — 短期 通过 + 长期 主体黑名单 几乎不可逆
- **不要 跟 接入商 客服 谈 法律意见** — 客服 答疑 限于 平台流程, 法律 / 监管 / 政策解读 找 律所

## 4. 工作流 (4 个 完整 SOP, 每个含 入门 vs 资深 显式 diff)

### 4.A. ICP 主体 + 网站 首次备案 (非经营性 通用版)

**入门 SOP** (代办 + 标准流程, 21-30 工作日):

1. **主体证件** (个人: 身份证; 企业: 营业执照 + 法定代表人 身份证) + **域名实名认证** (注册商 同主体名) — 提前 3 工作日 确保 域名 状态 已实名
2. 在 接入商 (云厂商 / IDC) 备案系统 (例: help.aliyun.com) 创建 备案账户 → 上传 主体证件 + 域名 + 网站负责人 (个人 = 申请人; 企业 = 法定代表人 或 授权 网站负责人) + 网站基本信息 + 接入信息 (服务器 IP)
3. 接入商 初审 (1-3 工作日) → 通过 → 进入 通管局 审核队列
4. **短信核验** (备案手机号收到核验短信, 24h 内回复 完成核验) + 部分省份 **人脸核验** (例如 沪 / 京 / 粤 通管局)
5. 通管局 审核 (1-20 工作日 — 沪 / 京 / 粤 平均 7-10; 中西部 平均 12-18 业内估) → 通过 → 取得 ICP 备 XXXXX-N 号
6. 接入商 通知 备案成功 → **30 工作日内** 网站底部 必须 添加 ICP 备案号 + 公示链 (beian.miit.gov.cn 跳转链)
7. 同步 30 日内 启动 公安备案 (SOP 4.B)

**资深路径** (≥ 3 项 显式 diff):

- **跳过** 短信核验 单一渠道焦虑 — 资深者 一次同时 备案 5+ 站 (一个主体下 多个网站 共用同一备案手机号), 批量 短信验证 一次性 处理
- **优化** 域名实名 + 网站建设方案 同步 — 资深者 在 域名注册 当下 直接 用 备案主体名 注册, 域名实名 已经先行, 备案启动时 域名已 ready, 跳过 3-7 工作日 等待
- **额外** 接入商网络 + 通管局 选型 配合 — 资深者 根据 业务实际所在 + 该省通管局 审核速度 选 主体所属省份 (新设主体 注册时 即设计), 不要 上海主体 全国接入 (跨省 接入信息变更 麻烦)
- **额外** 多接入备案 备份 — 资深者 同一主体 在 阿里云 + 腾讯云 都接 (两个 接入号 在 同一备案下) 防 单一云厂商 故障; 但 注意 主备案与子备案区分

**常见 驳回 + 应急**:

- 域名 与 主体不一致 → 重新做 域名实名 (注册商) 或 域名转移 (3-7 工作日)
- 网站负责人 信息不真实 → 重传 身份证 + 在职证明 (企业)
- 接入商 信息错填 → 接入商客服 协助 修改 后 重提
- 通管局 退回 (原因: 主体类型 / 内容描述 / 域名 / IP 不匹配) → 按 退回原因 整改 后 重提 (不计入 抽查 黑历史)

### 4.B. ICP 经营许可证 申请 (B25 信息服务业务 — 经营性 必备)

**入门 SOP** (标准代办流程, 60-90 工作日):

1. **公司基本条件准备**: 注册资本 (跨地区 1000 万 / 省内 100 万 业内估) + 经营范围 含 "增值电信业务" + 公司 3 名以上 缴纳社保 ≥ 1 月 (高级管理人员) + 接入资源 (云厂商 + 接入合同)
2. 准备 申请材料 (公司证照 + 法定代表人 身份证 + 验资报告 / 银行流水 + 接入合同 + 业务发展计划 + 信息安全保障措施 + 站点 详细业务描述)
3. 提交 工信部 综合管理平台 (跨地区) 或 各省通管局 (省内) → 形式审核 (1-5 工作日) → 接收
4. 实质审核 (50-60 工作日) → 通过 → 颁发 ICP 经营许可证
5. 拿证后 同步 完成 ICP 备案 (如有) + 公安备案 + 等保 (若 需) + 算法 / 大模型 备案 (若 需)

**资深路径** (≥ 3 项 显式 diff):

- **优化** 主体设计 — 资深者 提前 6-12 月 注册 子公司 (集团 直接 申请 是 历史包袱; 新设 干净 主体 路径快); 跨地区 子公司 在 工信部 直接 申请 比 各省通管局 路径 整体 30-45 天 快
- **跳过** 自己写 业务描述 + 信息安全制度 — 资深者 用 律所 + 备案服务 套用 工信部 偏好 模板 + 反复 改 4-5 版, 不是 自己 凭空写 (退回率 节省 40%+ 业内估)
- **额外** 接入商 同步 配合 — 资深者 在 提交申请 同时 已经 锁定 接入商 + 签接入合同 + 部署 测试环境, 拿证后 1 周内 上线; 不要 拿证再找接入
- **额外** 跨省运营 提前 规划 — 经营性 ICP 跨省 需 工信部 (不只 单省通管局); 业务 拓展 5 个省以上 直接 走 跨地区 申请, 不要 逐省 累加

**常见 驳回 + 应急**:

- 注册资本 不实 (虚高 / 短期 抽逃) → 重做 验资 + 银行 长期 流水 证明
- 业务描述 与 实际 不符 / 含 高敏感 业务 (政治 / 涉黄 / 涉赌 / 跨境 直接) → 重写 业务范围 + 删 敏感 模块
- 接入资源 不充足 → 补 多家 接入商 接入合同
- 3 名社保 未到 1 月 → 等 1 月再提

### 4.C. APP 备案 (工信部 2024-03 新规)

**入门 SOP** (主体 + APP 完整路径, 15-30 工作日):

1. **前置**: APP 主体 已完成 ICP 备案 (如未备案, 先 走 SOP 4.A)
2. APP 主体 在 接入商 (例 阿里云 / 腾讯云) APP 备案 系统 创建 备案账户 → 上传 APP 信息 (名称 / 包名 / 版本号 / 平台 iOS / Android / HarmonyOS / 小程序 / 快应用 / 应用商店分发地址)
3. 接入商 初审 (1-3 工作日) → 提交 通管局
4. **短信 + APP 实测** (通管局可能 实际 下载 / 测试 APP 是否符合 备案描述)
5. 通管局 审核 (7-15 工作日) → 通过 → 取得 APP 备案号 (与 ICP 备案号 关联)
6. **上线前 30 工作日内** APP 必须 在 各 应用商店 上架页 标注 备案号; 自 2024 年起 应用商店 全面下架 未备案 APP

**资深路径** (≥ 3 项 显式 diff):

- **跳过** 单一 接入商 限制 — 资深者 同一 APP 在 多个 接入商 都备案 (主备 关系), 防 单一 云厂商 故障 + 多渠道 上架; 但 注意 APP 主备案 必须 单一 主体
- **优化** APP 包名 + 主体 同步 — 资深者 APP 上线规划阶段 即统一 包名 (com.{主体英文名}.{产品}) + 主体名 + 法定代表人 一致, 不要 APP 备案时 才发现 包名 历史包袱
- **额外** 小程序 + HarmonyOS 应用 同步备案 — 资深者 主 APP 备案时 一并 备案 微信小程序 + HarmonyOS 应用 (新规 涵盖 5 类 应用 — iOS / Android / HarmonyOS / 小程序 / 快应用)
- **额外** 跨境业务 提前 APP 主体 国内 落地 — 海外母公司 APP 在 中国 上架 必须 国内主体 备案 (子公司 + WFOE), 不要 临时 找 代理 公司 走 备案 (主体不一致 高风险)

**常见 驳回 + 应急**:

- APP 包名 与 备案信息 不一致 → 重新 提交备案 (含改 包名)
- APP 内容 与 备案描述 不符 (例如 备案 "工具", 实际 "社交") → 整改 APP 内容 或 重新 备案 + 升级类型
- 小程序 / HarmonyOS 应用 未一并 备案 → 单独 补办 (但 接入商 不必新设)

### 4.D. 数据出境 安全评估 申报 (CAC, 重要数据 / 大量个人信息)

**入门 SOP** (CAC 评估路径, 60-180 工作日):

1. **触发条件 自查**: (a) 处理 100 万人 + 个人信息; 或 (b) 累计 出境 100 万人 + 个人信息 + 1 万人 + 敏感个人信息 (年度); 或 (c) 重要数据 出境; 或 (d) 关键信息基础设施运营者 (CIIO) 出境 任何 个人信息 / 重要数据 — 任一 触发 必须 走 CAC 安全评估 (不能 走 标准合同)
2. **自评估**: 数据接收方 国家 / 地区 数据保护水平 + 出境必要性 + 出境数据 范围 + 接收方 处理目的 / 方式 / 期限 — 写 自评估报告
3. 提交 各省网信办 → 转报 国家网信办 (CAC) → CAC 受理 (7 工作日 决定 是否受理)
4. CAC 实质评估 (45-120 工作日) → 通过 → 出具 评估结果 (有效期 通常 2 年; 重大变化 须 重新评估)
5. 通过后 出境 + 持续 监测 + 年度自查; 评估结果 失效 前 90 日 启动 重新评估

**资深路径** (≥ 3 项 显式 diff):

- **跳过** 自己写 自评估报告 — 资深者 用 大型 cyber 律所 (君合 / 中伦 / 金杜 / 大成) 操作 — 律所 知 CAC 偏好 + 模板; 自己 写 退回率 高
- **优化** 数据出境 地图 + 必要性 论证 — 资深者 提前 6 月 梳理 数据流 + 接收方 + 类目 + 必要性, 评估申请前 已有 完整 数据出境地图; 不是 边申请 边补
- **额外** 接收方 配套 国家 / 地区 数据保护 资质 — 资深者 在 接收方 (海外母公司 / 海外SaaS / 海外 数据中心) 提前 取得 GDPR DPA / Privacy Shield successor (DPF) / 当地 数据保护 资质, 增强 评估通过率
- **额外** 多路径 并行 — 资深者 把 不需 CAC 评估 的 数据 走 标准合同 + 需 评估 的 走 评估, 不要 全部 进 CAC; 评估周期 长 + 不通过 风险, 拆解 路径 灵活 重要

**常见 驳回 + 应急**:

- 接收方 国家 数据保护水平 不足 → 升级 接收方 资质 或 改 路径 (例如 接收方 转 数据本地化 处理, 不出境)
- 必要性 论证 弱 → 重写 必要性 + 业务影响 + 替代方案 评估
- 数据范围 过宽 → 重新 限定 + 缩窄 + 改 评估申请

## 5. 表达 DNA (Voice — 4 类 各 2-3 段)

### 5.1 严肃 监管解读 (学者 / 律所 / 智库 — 正式 + 法学术语 + 谨慎 hedging)

> 「《非经营性互联网信息服务备案管理办法》第 4 条 规定 非经营性互联网信息服务 提供者 应当 履行 备案手续, 这是 单一 强制性 行政备案 — 申请 + 通过 + 上线 不存在 选择性; 同时 第 11 条 设定 接入服务者 信息真实性核验义务 与 异常 上报义务, 形成 国务院令 292 号 + 信部令 33 号 双轨 责任 闭环。学者 (周汉华, 2021) 与 多家律所 (君合 / 中伦 cyber alerts) 均指出, 实务 中 接入商责任 是 中国独有 的 ICP 备案 制度 不同 海外 'register & forget' 的 关键支柱, 不可 简单 类比。」 [一手 引 / 立法 文本 + 学术 + 律所]

> 「数据出境 安全评估 申报 路径 (cac.gov.cn 数据出境安全评估办法 2022 第 4 条 三类 触发 + 第 5 条 自评估 + 第 9 条 受理 + 第 12 条 评估期限 45 工作日 可延 60 日) 在 滴滴 80.26 亿 罚款 (CAC 通报 2022-07-21) 后 由 选择性走 转向 强制 全栈, 接收方 国家 数据保护水平 + 业务必要性 + 数据范围 缩窄 三要素 缺一不可。」 [一手 引 / 法规 条文 + 监管 通报]

### 5.2 业内 实务 (备案代办 / 接入商 备案 PM / 站长 — 直白 + 流程 step-by-step + 偏务实)

> 「这事儿其实没那么玄。你就两步: 第一, 上 阿里云备案后台 (或腾讯云 / 华为云, 你接入哪家就在哪家), 把 营业执照 + 法定代表人 + 网站负责人 + 域名 + 服务器 IP 这五件套填了; 第二, 等 接入商 客服 给你 打电话 / 发短信 核验, 你 别 错过 就行。前后 两个礼拜搞定 — 上海 7-10 天, 二三线 12-15 天 业内估。卡的最多的是 域名实名 没做完, 或者 个人主体 想做 电商, 这两种 直接 退回。」 [典型 接入商 备案 PM 答疑 风格]

> 「公安备案 ICP 备案 不一样 别搞混 — ICP 是 beian.miit.gov.cn (工信部), 公安是 www.beian.gov.cn (公安部), 两个网站两个备案号, 不互通。ICP 备完 30 天内 必须 公安备案, 你不办 公安部 抽查 直接 关你站。安全员 找个 内部 IT 兼任, 联系电话 24h 通畅。」 [常见 站长 互助 风格]

### 5.3 学术 + 立法 视角 (法学院教授 / 立法专家 — 概念辨析 + 制度比较 + 法理推理)

> 「区分 行政许可 与 行政备案 是 中国 互联网监管 制度设计 的 第一公理。ICP 备案 属于 行政备案 (备案制), 程序 上 是 申请 + 通过 + 上线, 法理 上 是 形式 审查 + 推定 合法; ICP 经营许可证 属于 行政许可 (许可制), 程序 上 是 申请 + 实质 审查 + 颁证, 法理 上 是 设定 + 信赖 利益 + 撤销 救济; 业内 长期 把 这两词 混用 是 法律错位 的 典型表现。立法层面, 《行政许可法》(2003) 与 《行政许可法实施条例》(2008) 是 总章, 国务院 / 部委 规章 是 具体设定, 任何 互联网监管 合规 第一步 都是 判断 你的业务 落 哪一类。」 [典型 法学院 教授 课堂 / 论文 风格]

> 「个人信息保护法 PIPL (2021) 域外效力 (第 3 条) 与 跨境数据流动 三路径 (第 38 条 — 安全评估 / 标准合同 / 认证) 是 中国 数据合规 与 GDPR + CCPA 接轨 同时 体现 中国特色 的 标志性 制度安排。在 自评估 报告 中, 接收方 数据保护水平 不仅 比照 国家 立法, 还须 考量 接收方 内部 数据保护实践 + 处理目的 限定 + 接收方 提供 救济 路径, 这部分 实务 操作 (CAC 安全评估办法 2022 + 标准合同办法 2023) 与 学术 (张新宝, 2022; 程啸, 2023) 共识 是 实质 评估 重于 形式 合规。」 [典型 个保法 学者 风格]

### 5.4 反面 + 警示 (监管 处罚 通告 + 业内 反思 — 严控 不软化 + 警示性 + 具名 案例)

> 「滴滴出行 因 违反 《网络安全法》《数据安全法》《个人信息保护法》, 在 2022-07-21 被 国家互联网信息办公室 通告 处以 人民币 80.26 亿元 行政处罚 (CAC 通报 原文), 同时 主要负责人 各处罚 100 万元。本案 是 中国 数据合规 史上 第一例 80 亿元级别 处罚, 也是 网信办 + 工信部 + 公安部 + 国家市场监督管理总局 + 央行 + 国家税务总局 多机构 联动 执法 的 标志性案例。教训: (a) 数据 出境 安全评估 不是 选择题 — 触发 任一 条件 必走 评估, 不存在 'crash 着先上, 出事再说'; (b) 主要负责人 个人罚款 (100 万元) 不在 公司罚款 之外 是 附加, 在 法人 + 主要负责人 双层 责任体系下 不可分离; (c) 关键信息基础设施 运营者 出境 任一 个人信息 都需 评估 — 这是 关基条例 + PIPL + DSL 三法 联动 的 强制规则, 任何 主张 '我只是网约车不是关基' 的 抗辩 都 无效。」 [CAC 通报 + 律所 alert 风格, 严控 不软化]

## 6. 质量基准 + 反模式

1. **三横三纵 九宫格 完整** — 任何 接到的 中国互联网项目 必须 在 5 分钟内 画出 备案 + 公安 + 等保 × 主体 + 网站 + 经营性升级 九宫格 + 每格 现状; 任一格 标 "不清楚" 必须 先停 (业内估 70% 项目 在 该步 暴露 风险)
2. **接入商 vs 自办 区分清晰** — 任何 站长 来问 "我能不能 自己 直接 向工信部 提交?" 必须 直接 答 "中国独有 接入商责任制, 不能", 不存在 第三选项
3. **行政许可 vs 行政备案 不混淆** — 任何 法律分析 中, 经营性 / 非经营性 区分 + 备案制 / 许可制 区分 必须 第一句话 给出; 学术 + 律所 + 监管 三方 共识
4. **各省口径差异 主动提示** — 任何 通用 SOP 必须 标注 "沪 / 京 / 粤 严; 中西部 松" + 备选 主体所属省份 (新设主体) 三选项, 不是 笼统 "中国"
5. **数据出境 + 跨境 主动联动** — 任何 涉及 海外母公司 / 海外 SaaS / 跨境数据 的 项目 必须 主动 提出 PIPL 第 38 条 三路径 + 接收方 资质 评估; 不是 用户 问了 才答

### 6.2 反模式 (10 条)

1. **海外服务器 想绕开备案** — DNS GFW 污染 + 国内 ICP 检查 自动屏蔽 + 速度极慢, 短期方便 长期 不可行
2. **个人备案 做商业站** — 工信部抽查 + 强制注销 + 接入商暂停 + 半年 不能再备案
3. **找黑代办 走灰色通道 (假资料)** — 主体 5 年内 黑名单 + 公安行政拘留, 几乎 不可逆
4. **备案后 内容大幅偏离 备案描述** — 工信部 + 网信办 抽查 强制注销 + 罚款
5. **云厂商 备案 与 服务器 不匹配** — 接入商核验拒绝 + 工信部抽查注销
6. **公安备案 忽略 / 拖延** — 公安部 + 网安总队 通报 + 严重 关停
7. **等保 找便宜测评 + 应付式过场** — 公安部 抽检 通报 + 招投标 资质受限
8. **数据出境 忽略 / 简单走 标准合同 不评估 数据敏感度** — CAC 巨额罚款 (滴滴前车之鉴)
9. **算法 / 大模型 服务 未备案 直接上线** — 责令下架 + 罚款 + 严重 永久禁入
10. **把 ICP 备案 = 整个互联网合规** — 任何 中型以上站点 必须 ICP + 公安 + 等保 + 算法 + 数据出境 + 个保 + 行业特许 多栈联动, 缺一不可

## 7. 智识谱系 (流派 / 学派)

中国互联网监管 + 内容合规 法学 + 实务 主要分 9 个流派/中心:

1. **中国社科院 法学所 网络与信息法 学派** — 周汉华 主导, 立法核心 + 政策建议; 个保法 + 数安法 立法专家组 召集人; 最受 中央 + 国务院 重视, 是 立法第一线 学派
2. **人大法学院 民法 + 个保法 学派** — 王利明 + 张新宝 主导, 民法典 网络相关 章节起草 + 个保法 学术深度; 与 司法 实务 联系紧密
3. **清华法学院 数据法 + 个保法 学派** — 程啸 主导, 个保法 + 数据法 学术专著; 偏向 比较法 + 制度设计
4. **北大法学院 行政法 + 数据合规 学派** — 王锡锌 主导, 行政法视角看数据合规; 与 司法 + 监管 实务 联系
5. **华东政法 (华政) 数据要素 + 数据出境 学派** — 高富平 + 沈伟 主导, 数据要素 市场 + 数据出境 跨境业务; 偏 国际化 + 产业应用
6. **北航法学院 网络空间法律 + 一带一路 学派** — 龙卫球 主导, 网络空间法律 + 一带一路 数据治理; 偏 比较法 + 国际治理
7. **对外经贸 法学院 数据合规 学派** — 许可 主导, 数据合规 + 跨境业务; 偏 跨国公司 + 国际业务
8. **TMT / cyber 律所 实务派** — 君合 (郑娟 / 葛江虬) / 中伦 (童甲 / 王智斌) / 金杜 (王军 / 苗瑾秋) / 大成 / 海问 / 汉坤 / 锦天城 等 主导, 客户警示 alerts 是 政策模糊期 唯一公开 实务指引

核心分歧:
- **学术 (立法) vs 律所 (实务)** — 学术 长期 关注 制度设计 与 法理; 律所 长期 关注 客户 实际操作; 重大政策 出台 前 学术 影响立法, 出台 后 律所 解读 + 实务跟进
- **比较法 学派 (高富平 / 龙卫球 / 许可) vs 制度内部 学派 (周汉华 / 张新宝 / 程啸)** — 比较法 关注 GDPR / CCPA 接轨; 制度内部 关注 中国特色 + 自主路径

## 8. 诚实边界 (9 条)

1. 本 skill 是 备案合规 + 互联网内容合规 + 行业 OS + 决策框架, 不替代 律师 个案 法律意见 + 监管 实际 沟通 + 实际 提交 操作
2. 信息截止 2026-05, 中国 互联网监管 立法 + 备案系统 + 各省执行口径 + 工信部抽查节奏 + 大模型 / 算法 备案 6-12 月内 可能 有 1-3 次 实质性 微调; 关键决策 前 必须 二次 核实 .gov.cn 原文 + 律所 最新 alerts
3. 公开材料偏差 — 中国 监管原文 + 律所 alerts 一手 充足 (本 prototype 一手率 99.7%, 历史最高); 但 (a) 各省通管局 实操口径 差异 极大 且 不公开 (沪 / 京 严, 二三线 松, 差异 ≥ 30% 业内估); (b) 工信部 / 网信办 / 公安部 内部 抽查节奏 + 处罚梯度 不公开; (c) 失败案例 (备案被拒 / 注销 / 处罚) 当事方 多 沉默; (d) 中国 互联网协会 + 中国信通院 智库报告 多偏 政策解读, 实操细节 一笔带过
4. 区域差异 — 各省通管局 执行口径 差异 极大, 同样 主体 + 内容 在 不同省 备案通过率 与 处罚力度 差异 30%+ 业内估; 本 skill 给的 入门 SOP 是 通用 最严 执行版本 (基于 沪 / 京 / 粤 三省 主流口径), 二三线 省份 可能 略宽松
5. 强监管 + 政治属性 — 本 skill 严格 限定 在 合规 + 工程 + 法务 视角, 不涉 政治 + 内容审查 价值判断; 任何 推荐 都包含 隐含的 不评论 政治 假设
6. 外资 / 跨境部分 (VIE / 外商投资 / 数据出境 / 海外 SaaS 落地) 实务上 受 中央政策 + 国际关系 影响 极大, 不可 仅凭 法律字面 推断; 任何 跨境决策 必须 律所 个案意见
7. AI / 算法 备案 实证落地 (大模型 服务备案 + 算法备案 + 深度合成 备案) 仍在 高速 演变中 (2023-2026), 各省 + 网信办 + 工信部 联动 节奏 不稳定; 本 skill 给的 框架 是 截止 2026-05 状态, 6 月以上 hype 衰减 需 二次 核实
8. 反面案例 (滴滴 / 知网 / 黑代办 / 字节 等) 全部 引 一手 监管通告 (CAC / SAMR / 工信部 / 公安部) 原文, 案例视角 NOT 个人攻击; 但 任何 案例 都有 时代局限 — 滴滴 是 2022 中美 数据 + 国家安全 紧张期 的 标志性案例, 当下 大模型 案例可能 替代 成为 新 anchor
9. 本 skill 不收录 翻墙 / VPN / 海外 dynamic IP 等 违法绕过 内容, 也不 评估 任何 "灰色地带" 实操可行性 — 这些 即使 短期 实际 可行 长期 都是 高 政策风险, 与 master OS 的 持续合规 假设 完全相反
