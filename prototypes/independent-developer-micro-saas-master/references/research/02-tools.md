## 必备层 (13 件, 约 80%+ 独立开发者使用)

### 1. Next.js

- **One-liner**: React 全栈框架，最广泛的独立开发者 SaaS 起点
- **Layer**: 必备
- **Category**: framework
- **URL**: https://nextjs.org
- **Pricing**: 开源免费；托管在 Vercel 有 free tier，Pro 从 $20/mo
- **适合**: 前端 React 开发者、需要 SSR/SSG/API routes 一体的全栈 SaaS、与 Vercel 生态深度整合
- **不适合**: 非 JS 技术栈偏好者；对 Vercel 有强烈供应商锁定顾虑者；pure API 后端
- **替代品**: SvelteKit (更轻量), Remix (更 web standards), Rails (Ruby 全栈)
- **独立开发者视角**: MicroConf State of Independent SaaS 调查显示 Next.js 是独立 SaaS 最常用框架之一；与 Supabase + Stripe + Vercel 组合形成 "modern indie stack"，大量 starter kit 和教程资源降低了从零开始的摩擦。evidence: [T02-S028][T02-S029][T02-S004]
- **Endorsement evidence**: IndieHackers 常见栈选择 [T02-S029]；Starter Story 多家 SaaS 使用 [T02-S030]
- **可信度**: 高 | **Decay risk**: 低 — App Router 已稳定，生态持续扩大

---

### 2. Stripe

- **One-liner**: 支付处理 + 订阅账单基础设施，独立开发者默认选择
- **Layer**: 必备
- **Category**: payment
- **URL**: https://stripe.com
- **Pricing**: 2.9% + $0.30/transaction；无月费；Billing/Radar 附加功能按量计费
- **适合**: 全球支付（135+ 货币）；B2B SaaS 订阅；需要 webhooks + SDK 的开发者；一次性付款和订阅混合
- **不适合**: 需要 VAT/GST/销售税完全代收代缴的独立开发者（用 LemonSqueezy 或 Paddle）；中国大陆主要用户群
- **替代品**: LemonSqueezy (MoR + 税务代管), Paddle (MoR 企业级), Gumroad (极简数字产品)
- **独立开发者视角**: 业界标准，采用率估计 80-90%+。API 质量和文档被普遍认为是行业最佳。主要缺点是税务责任仍在卖家一侧——全球 VAT/GST 合规需额外处理。evidence: [T02-S001][T02-S028][T02-S029]
- **Endorsement evidence**: MicroConf 调查中最常被提及支付工具 [T02-S028]；patio11 在 Stripe 任职并公开背书最佳实践 [T02-S011]
- **可信度**: 极高 | **Decay risk**: 极低

### 3. Supabase

- **One-liner**: 开源 Firebase 替代品：Postgres + Auth + Storage + Edge Functions + Realtime
- **Layer**: 必备
- **Category**: database
- **URL**: https://supabase.com
- **Pricing**: Free tier (2 个项目, 500MB DB)；Pro $25/mo；用量超出按量计费
- **适合**: 需要托管 Postgres + 认证一体方案；不想分开管 DB 和 Auth；需要 Row Level Security
- **不适合**: 超大规模生产环境（free tier 暂停规则需注意）；非 Postgres 偏好；严格中国数据驻留要求
- **替代品**: Neon (serverless Postgres 更专注), PlanetScale (MySQL), Firebase (Google 生态)
- **独立开发者视角**: 2022-2025 爆发增长，成为 Next.js 生态系统最搭配的后端选择。Auth + DB 合一极大减少了独立开发者需要集成的工具数量。evidence: [T02-S003][T02-S029][T02-S028]
- **Endorsement evidence**: Supabase 自身博客有大量 indie dev 案例 [T02-S003]；IndieHackers 讨论中频繁出现 [T02-S029]
- **可信度**: 高 | **Decay risk**: 低 — 功能仍在扩展（向量/边缘函数）

### 4. Vercel

- **One-liner**: Next.js 最佳部署平台，零配置 CI/CD + CDN + 预览部署
- **Layer**: 必备
- **Category**: deployment
- **URL**: https://vercel.com
- **Pricing**: Free (个人/非商业)；Pro $20/mo/user；Enterprise 按需
- **适合**: Next.js/React 前端；需要自动预览部署；Serverless Functions；Edge Middleware
- **不适合**: 需要长时运行后台 job；有 Docker/容器化需求；成本敏感的高流量应用（带宽费贵）
- **替代品**: Netlify (类似 Jamstack), Cloudflare Pages (更便宜带宽), Railway (更通用后端)
- **独立开发者视角**: Next.js 框架由 Vercel 开发，两者深度整合。对于独立开发者，零配置 CI/CD 显著减少 DevOps 负担。但流量规模增大后费用增长快，需关注。evidence: [T02-S002][T02-S028]
- **Endorsement evidence**: Next.js 官方文档推荐 Vercel 部署 [T02-S004]；MicroConf 调查中高频出现 [T02-S028]
- **可信度**: 高 | **Decay risk**: 低

### 5. Clerk

- **One-liner**: 为 Next.js/React 优化的完整认证即服务，含社交登录 + MFA + 组织管理
- **Layer**: 必备
- **Category**: auth
- **URL**: https://clerk.com
- **Pricing**: Free (10k MAU)；Pro $25/mo；增长计划按 MAU 计费
- **适合**: Next.js App Router；需要快速集成完整 auth UI；B2B multi-tenant 需要 Organizations
- **不适合**: 纯 API 后端（无 UI）；极度成本敏感的高 MAU 产品；已用 Supabase Auth 的项目
- **替代品**: Supabase Auth (已含在 Supabase 中), Auth0 (老牌企业级), NextAuth.js (开源自托管)
- **独立开发者视角**: 2023-2025 快速成为独立开发者 auth 首选。UI 组件开箱即用，与 Next.js App Router 深度整合。相比 Supabase Auth 功能更丰富，相比 Auth0 更开发者友好。evidence: [T02-S005][T02-S029]
- **Endorsement evidence**: IndieHackers 技术栈讨论中高频出现 [T02-S029]；多个 SaaS starter kit 采用 [T02-S004]
- **可信度**: 高 | **Decay risk**: 低

### 6. Cursor

- **One-liner**: AI-first IDE，显著加速独立开发者编码速度，2024 年最热门工具
- **Layer**: 必备
- **Category**: ai-coding
- **URL**: https://cursor.com
- **Pricing**: Free (限量)；Pro $20/mo；Business $40/user/mo
- **适合**: 所有使用代码编辑器的独立开发者；特别适合快速原型、不熟悉的代码库、boilerplate 生成
- **不适合**: 极端隐私/代码保密需求（代码上传服务器）；不愿付月费者可用 Copilot 替代
- **替代品**: GitHub Copilot (更保守价格), Windsurf (Codeium 出品), Aider (CLI 替代)
- **独立开发者视角**: 2024 年度独立开发者讨论最热工具。"10x 开发速度"的说法虽夸张但反映真实生产力提升。对单人团队尤其重要——AI 辅助可弥补缺乏 code review 的劣势。MicroConf 2024-2025 讨论中大量提及。evidence: [T02-S008][T02-S028][T02-S029]
- **Endorsement evidence**: 独立开发者社区讨论中高频推荐 [T02-S029]；MicroConf 状态调查显示 AI 工具采用率快速上升 [T02-S028]
- **可信度**: 高 | **Decay risk**: 中 — 竞争激烈，产品快速迭代

### 7. Tailwind CSS + shadcn/ui

- **One-liner**: Tailwind 提供 utility-first CSS，shadcn/ui 提供可复制组件库，无需设计师
- **Layer**: 必备
- **Category**: design
- **URL**: https://tailwindcss.com / https://ui.shadcn.com
- **Pricing**: 两者均开源免费
- **适合**: 独立开发者自己做 UI；需要快速构建专业外观 SaaS；React 技术栈
- **不适合**: 非 React 框架（shadcn/ui）；需要完整 Design System 的大型团队
- **替代品**: Bootstrap (老牌), Mantine (React), DaisyUI (Tailwind 变体)
- **独立开发者视角**: 组合是 2023-2025 独立开发者最标准的 UI 解决方案。shadcn/ui 的"复制粘贴而非安装"模式让开发者对代码有完全控制权，避免样式冲突和版本锁定。与 Cursor 结合后，AI 能精准生成组件。evidence: [T02-S012][T02-S013][T02-S029]
- **Endorsement evidence**: IndieHackers stack 讨论中标配 [T02-S029]；大量 SaaS starter kit 内置 [T02-S004]
- **可信度**: 极高 | **Decay risk**: 低

### 8. Resend

- **One-liner**: 开发者优先的事务邮件 API，用 React 写邮件模板
- **Layer**: 必备
- **Category**: email
- **URL**: https://resend.com
- **Pricing**: Free (3k emails/mo)；Pro $20/mo (50k emails)；按量计费
- **适合**: 需要发送注册确认/密码重置/通知邮件；用 React 写模板（React Email）；快速集成
- **不适合**: 营销邮件群发（用 ConvertKit/Loops）；超大规模高频邮件（用 Postmark/SendGrid）
- **替代品**: Postmark (高送达率老牌), SendGrid (企业级大量), AWS SES (最便宜但配置复杂)
- **独立开发者视角**: 2023 年后快速成为独立开发者事务邮件首选。与 React/Next.js 生态完美契合，React Email 让邮件模板维护成本极低。evidence: [T02-S010][T02-S029]
- **Endorsement evidence**: IndieHackers 社区反复出现 [T02-S029]；shadcn/ui 相关 starter 默认使用 [T02-S013]
- **可信度**: 高 | **Decay risk**: 低

### 9. Plausible Analytics

- **One-liner**: 隐私优先的 Google Analytics 替代品，无 Cookie，GDPR 合规
- **Layer**: 必备
- **Category**: analytics
- **URL**: https://plausible.io
- **Pricing**: $9/mo (up to 10k pageviews)；按流量层级计费；可自托管免费
- **适合**: 欧洲用户群为主；不想处理 Cookie consent banner；看关键指标够用
- **不适合**: 需要 Google Ads 转化追踪；需要 cohort analysis / funnel 深度分析；免费
- **替代品**: Fathom Analytics (类似隐私优先), PostHog (开源 + 产品分析), Google Analytics 4 (免费但复杂)
- **独立开发者视角**: Plausible 本身就是独立开发者创业案例（Marko Saric + Uku Taht 在 37Signals 之前两人团队）。其博客是独立开发者社区的一手来源。对于欧洲 SaaS，避免 GA 的 GDPR 风险很重要。evidence: [T02-S009][T02-S030][T02-S029]
- **Endorsement evidence**: Plausible 博客本身记录了 bootstrapped 增长过程 [T02-S009]；IndieHackers 社区隐私分析讨论中常被推荐 [T02-S029]
- **可信度**: 极高 | **Decay risk**: 低

### 10. ConvertKit (Kit)

- **Layer**: 必备

### 11. Cloudflare

- **One-liner**: CDN + DNS + DDoS 防护 + Workers (serverless) + R2 (存储)，免费层极为慷慨
- **Layer**: 必备
- **Category**: deployment
- **URL**: https://cloudflare.com
- **Pricing**: Free tier（CDN/DNS/Pages/R2 基础）；Pro $20/mo；Workers 按调用量计费
- **适合**: DNS 管理；免费 CDN；Workers 运行轻量边缘逻辑；R2 作为 S3 的 0 出流量费替代

- **适合**: 想出售 $50k-$5M ARR 区间 SaaS；需要流动性退出但不到 PE 级别；购买已验证业务
- **不适合**: 超大规模退出（需传统 M&A）；未盈利早期项目

- **独立开发者视角**: Acquire.com（原 MicroAcquire，由 Andrew Gazdecki 创立）把小型 SaaS 的退出市场正规化了。对于独立开发者，这提供了一个之前不存在的流动性选择。evidence: [T02-S037][T02-S029]
- **Endorsement evidence**: IndieHackers 退出案例分享中常被提及 [T02-S029]；Starter Story 含退出相关采访 [T02-S030]
- **可信度**: 高 | **Decay risk**: 低

## 工具栈选型决策树

```
你是哪种独立开发者？
│
├─ JS/TS 全栈 → Next.js + Supabase + Vercel + Clerk + Stripe + Tailwind/shadcn
│   └─ 需要 MoR (税务代管)？→ 换 Stripe 为 LemonSqueezy/Paddle
│
├─ Python/Ruby/Go 后端 → 自选后端框架 + Railway/Fly.io + Neon/PlanetScale + Stripe
│   └─ Rails 全栈？→ Rails + Kamal (自部署) 或 Render/Fly.io
│
├─ 移动 (iOS/Android) → React Native / Flutter + RevenueCat + Railway 后端
│
├─ 纯 API/工具型 SaaS → Next.js API Routes 或 FastAPI + Cloudflare Workers (边缘)
│
└─ 极简产品 (数字下载/会员) → Gumroad / LemonSqueezy + Buttondown 即可
```

**决策分支 1 — 支付选择**:
- 不想管税务 + 面向全球 → **LemonSqueezy** (早期) 或 **Paddle** (成长期)
- 愿意自己管税务 + 需要低费率 → **Stripe**
- 移动 IAP → **RevenueCat** + App Store/Google Play

**决策分支 2 — 数据库选择**:
- 需要 Auth + DB 一体 → **Supabase** (Postgres 全套)
- 纯 DB，Serverless 优先 → **Neon** (Postgres) 或 **PlanetScale** (MySQL)
- 边缘计算场景 → **Turso** (SQLite) 或 **Cloudflare D1**
- 已有 Rails/Django 经验 → 直接 Railway/Fly.io 上跑 Postgres

**决策分支 3 — 部署选择**:
- Next.js 项目 → **Vercel** (首选) 或 Cloudflare Pages
- 需要后端长时进程 / Docker → **Railway** 或 **Fly.io**
- 高流量成本优化 → **Cloudflare Workers** (边缘) + **R2** (存储)

## 避坑清单 (外行容易踩的坑)

1. **忘记 VAT/GST 合规直接用 Stripe**：Stripe 只是支付处理，税务责任在你。面向欧盟用户卖 $1k+/mo，应该用 LemonSqueezy/Paddle 作为 MoR，或者接受聘用税务顾问的成本。

2. **过早上 AppSumo**：LTD 用户的终身支持负担会拖累未来增长。一次 AppSumo 活动后，你可能有 1000+ 用户需要永久支持但不付续费。除非产品已稳定且刻意策略性使用，否则谨慎。

3. **用 Vercel 但忘记带宽费**：Vercel 的带宽费用在流量突增时会产生惊人账单。高流量场景考虑在 CDN 层加 Cloudflare 缓存，或迁移静态资源到 Cloudflare R2。

4. **用 Supabase Free Tier 做生产项目但不了解暂停规则**：Supabase 免费项目在 7 天不活跃后会暂停数据库。生产项目必须升级到 Pro。

5. **把事务邮件（注册确认）和营销邮件用同一发件域名**：用户退订营销邮件后可能影响事务邮件送达率。分开域名或发件人是最佳实践（Resend 做事务，ConvertKit/Loops 做营销）。

6. **早期过度工程化工具栈**：独立开发者最常见的错误之一是在有 10 个用户时就构建"可扩展架构"。Postgres + 一个简单后端往往比 Kafka + 微服务快 10 倍开发。

7. **认为 AI IDE (Cursor) 能替代架构思考**：Cursor 能加速代码实现，但无法替代好的产品决策和系统设计。技术债在 AI 辅助开发下可以积累得更快。

8. **Product Hunt 发布日不做准备**：Product Hunt 的算法在发布后 24 小时内特别重要。需要提前告知社区、准备好 Hunter、在发布日积极回复评论。没有准备的发布很少有好结果。

9. **中国出海者忽略 ICP 备案与境外合规双重约束**：面向全球的产品部署在境外服务器（如 Vercel/Cloudflare），但如面向中国大陆用户提供服务，ICP 备案仍可能被要求。两种合规路径不能简单混用。

10. **用 Intercom 但忽视成本曲线**：Intercom 随用户数线性涨价，独立开发者达到几千 MAU 后月费可达 $200-500+。早期用 Crisp（更便宜）或 Tawk.to（免费），等规模合理后再迁移。
